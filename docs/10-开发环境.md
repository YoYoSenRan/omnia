# 10 - 开发环境

## 前置依赖

| 工具 | 版本 | 说明 |
|------|------|------|
| Node.js | 22+ | 运行时 |
| pnpm | 9+ | 包管理器 |
| Docker | 24+ | 运行 PostgreSQL |
| Docker Compose | v2 | 编排本地服务 |

## Docker Compose

根目录 `docker-compose.yml` 提供 PostgreSQL 服务：

```yaml
services:
  postgres:
    image: postgres:16-alpine
    container_name: omnia-postgres
    restart: unless-stopped
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: omnia
      POSTGRES_PASSWORD: omnia
      POSTGRES_DB: omnia_core
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./scripts/init-db.sh:/docker-entrypoint-initdb.d/init-db.sh
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U omnia"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  postgres_data:
```

## 数据库初始化脚本

`scripts/init-db.sh` — 创建两个数据库（Docker entrypoint 自动执行）：

```bash
#!/bin/bash
set -e

psql -v ON_ERROR_STOP=1 --username "$POSTGRES_USER" <<-EOSQL
  CREATE DATABASE omnia_core;
  CREATE DATABASE omnia_app;
  GRANT ALL PRIVILEGES ON DATABASE omnia_core TO omnia;
  GRANT ALL PRIVILEGES ON DATABASE omnia_app TO omnia;
EOSQL
```

## 快速上手

```bash
# 1. 克隆项目
git clone <repo-url> omnia && cd omnia

# 2. 启动 PostgreSQL
docker compose up -d

# 3. 安装依赖
pnpm install

# 4. 初始化环境变量
cp apps/openclaw/.env.example apps/openclaw/.env
cp apps/server/.env.example apps/server/.env
cp apps/console/.env.example apps/console/.env
cp apps/web/.env.example apps/web/.env

# 5. 执行数据库迁移
pnpm --filter openclaw db:migrate
pnpm --filter server db:migrate

# 6. 启动开发
pnpm dev
```

## 数据库连接信息

| 数据库 | 用途 | 连接字符串 |
|--------|------|-----------|
| omnia_core | openclaw 服务 | `postgresql://omnia:omnia@localhost:5432/omnia_core` |
| omnia_app | server 服务 | `postgresql://omnia:omnia@localhost:5432/omnia_app` |

## 常用命令

```bash
# 启动/停止 PostgreSQL
docker compose up -d
docker compose down

# 重置数据库（删除数据重建）
docker compose down -v
docker compose up -d

# 连接数据库查看
docker exec -it omnia-postgres psql -U omnia -d omnia_core

# Drizzle Studio（可视化浏览数据库）
pnpm --filter openclaw db:studio
pnpm --filter server db:studio
```
