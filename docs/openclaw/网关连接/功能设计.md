# 网关连接 功能设计

文件：`src/gateway/`、`src/utils/env.ts`、`src/index.ts`

## 现状

- WebSocket 客户端已实现，支持 v3 协议握手
- 自动重连（指数退避，最多 10 次）
- 心跳机制（30s 间隔，3 次未响应则重连）
- RPC 请求/响应（Promise 化，10s 超时）
- 网关消息分发到对应 Service
- 配置来源仅限环境变量（GATEWAY_URL / GATEWAY_TOKEN）

## 一、配置发现（待实现）

### 1.1 设计思路

参考 MC 模式，采用**本地文件扫描 + 环境变量**双通道发现网关配置。openclaw 与 Gateway 通常部署在同一台机器上，直接读取 Gateway 的配置文件 `~/.openclaw/openclaw.json` 是最可靠的方式。

### 1.2 发现优先级

```
环境变量（最高） > openclaw.json 配置文件 > 默认值
```

| 配置项 | 环境变量 | 配置文件路径 | 默认值 |
|--------|---------|-------------|--------|
| Gateway URL | `GATEWAY_URL` | — | 从 host + port 拼接 |
| Gateway Host | `GATEWAY_HOST` | — | `127.0.0.1` |
| Gateway Port | `GATEWAY_PORT` | `gateway.port` | `18789` |
| Gateway Token | `GATEWAY_TOKEN` | `gateway.auth.token` 或 `gateway.auth.password` | — |
| Config 路径 | `OPENCLAW_CONFIG` | — | `~/.openclaw/openclaw.json` |

### 1.3 配置文件结构

`~/.openclaw/openclaw.json` 中与网关相关的字段：

```json
{
  "gateway": {
    "port": 18789,
    "mode": "local",
    "bind": "loopback",
    "auth": {
      "mode": "token",
      "token": "615aef524730..."
    }
  },
  "agents": {
    "defaults": {
      "model": { "primary": "provider/model-id" },
      "workspace": "~/.openclaw/workspace"
    },
    "list": [
      {
        "id": "main",
        "name": "main",
        "workspace": "~/.openclaw/workspace",
        "default": true
      }
    ]
  }
}
```

### 1.4 发现流程

```
新增 readGatewayConfig()
  ↓
读取 OPENCLAW_CONFIG（默认 ~/.openclaw/openclaw.json）
  ↓
解析 gateway.port / gateway.auth.*
  ↓
detectGatewayUrl()
  ├─ GATEWAY_URL 环境变量存在 → 直接使用
  └─ 否则 → ws://{host}:{port}
     ├─ host = GATEWAY_HOST || '127.0.0.1'
     └─ port = GATEWAY_PORT || config.gateway.port || 18789
  ↓
detectGatewayToken()
  ├─ GATEWAY_TOKEN 环境变量存在 → 直接使用
  └─ 否则 → 读取 config
     ├─ auth.mode = 'token'    → auth.token
     └─ auth.mode = 'password' → auth.password
```

### 1.5 Token 动态刷新

MC 的一个关键设计：每次重连时都重新读取配置文件获取最新 Token。Gateway 重启可能会生成新 Token，如果仍用旧值会导致认证失败。

```
重连触发
  ↓
scheduleReconnect()
  ↓
重新调用 detectGatewayToken()  ← 每次都读文件
  ↓
用最新 token 发起握手
```

### 1.6 实现位置

新增 `src/gateway/config.ts`：

```typescript
/** 从 openclaw.json 读取网关配置 */
readGatewayConfig(): GatewayFileConfig | null

/** 检测网关 URL（环境变量 > 配置文件推导） */
detectGatewayUrl(): string

/** 检测网关 Token（环境变量 > 配置文件） */
detectGatewayToken(): string
```

修改 `src/index.ts`：

```
// 改前
if (GATEWAY_URL) {
  new GatewayClient({ url: GATEWAY_URL, token: GATEWAY_TOKEN })
}

// 改后
const gwUrl = detectGatewayUrl()
if (gwUrl) {
  new GatewayClient({ url: gwUrl, tokenProvider: detectGatewayToken })
}
```

`GatewayClient` 构造函数新增 `tokenProvider` 选项（函数），在每次连接/重连时调用以获取最新 Token，替代固定的 `token` 字符串。

## 二、协议规范

### 2.1 帧格式

```typescript
interface GatewayFrame {
  type: "event" | "req" | "res"
  event?: string      // type=event 时的事件名
  method?: string     // type=req 时的 RPC 方法
  id?: string         // 请求/响应配对 ID
  payload?: unknown   // 请求参数或事件数据
  ok?: boolean        // type=res 时，成功标志
  result?: unknown    // type=res 时，响应数据
  error?: { message: string; code?: string }  // 错误信息
  seq?: number        // 事件序列号
}
```

### 2.2 握手流程

```
openclaw                           Gateway
   │                                  │
   │──── WebSocket 连接 ──────────────→│
   │                                  │
   │──── event: handshake ────────────→│
   │     {                            │
   │       protocol: "v3",            │
   │       client: "omnia-openclaw",  │
   │       mode: "server",           │
   │       token: <detected token>    │
   │     }                            │
   │                                  │
   │←──── res: ok ────────────────────│
   │                                  │
   │──── 心跳 ping (30s) ────────────→│
   │←──── pong ───────────────────────│
```

> Note: MC 浏览器端使用 challenge-response + Ed25519 签名，但我们是服务端到服务端通信，token 认证足够。

### 2.3 RPC 调用

```typescript
// 发起请求
client.request("agents.list", {})
// → 发送 { type: "req", method: "agents.list", id: "req_1", payload: {} }
// ← 等待 { type: "res", id: "req_1", ok: true, result: [...] }

// 超时 10s 后自动 reject
```

### 2.4 事件类型

| 事件 | 说明 | handler 处理 |
|------|------|-------------|
| `tick` | 网关心跳 tick（含 session 快照） | 仅日志 |
| `agent.status` | Agent 状态变更 | → agentService.updateStatus() |
| `task.created` | 新任务（网关侧创建） | → emitEvent() |
| `task.updated` | 任务状态变更 | → emitEvent() |
| `session.created` | 新会话 | → emitEvent() |
| `session.message` | 会话消息 | → emitEvent() |
| `log` | 日志条目 | 日志输出 |

## 三、连接管理

### 3.1 状态机

```
disconnected → connecting → connected
     ↑              │            │
     │              ↓            │
     └──────── error ←───────────┘
```

- `disconnected`：初始 / 主动断开
- `connecting`：WebSocket 建立中
- `connected`：握手成功
- `error`：连接失败或心跳超时

### 3.2 自动重连

| 参数 | 值 |
|------|-----|
| 最大重试次数 | 10 |
| 初始延迟 | 1s |
| 退避因子 | 1.7 |
| 最大延迟 | 15s |

公式：`delay = min(1000 × 1.7^attempt, 15000)`

连续 10 次重连失败后停止，状态变为 `error`。

重连时重新调用 `tokenProvider()` 获取最新 Token。

### 3.3 心跳

- 间隔：30s
- 发送 `ping` 帧，等待 `pong` 响应
- 连续 3 次未收到 pong → 判定连接死亡，触发重连

### 3.4 单例管理

```typescript
getGatewayClient(): GatewayClient | null  // 获取当前客户端
setGatewayClient(client): void            // 设置当前客户端
```

启动时 `detectGatewayUrl()` 返回非空则自动连接，否则 standalone 模式。

## 四、优化清单

### 4.1 已识别的问题

| # | 问题 | 影响 | 方案 |
|---|------|------|------|
| 1 | Token 来源只有环境变量 | Gateway 重启换 Token 后需重启 openclaw | 新增配置文件读取 + tokenProvider 动态刷新 |
| 2 | config 同步器不理解 `agents.list` 真实结构 | `discoverFromConfig()` 把 `config.agents` 当 flat array，实际是 `config.agents.list`，且每个 agent 的字段格式不同 | 修正解析路径，适配真实 `openclaw.json` 格式 |
| 3 | 缺少 Agent defaults 合并 | `agents.defaults.model` / `workspace` 未应用到各 agent | 同步时合并 defaults 到每个 agent |
| 4 | gatewayStatus API 信息不足 | 仅返回 connected/status，缺少 url、uptime 等 | 扩展返回 url / connectedAt / retryCount |
| 5 | 重连次数耗尽后无恢复手段 | 10 次失败后彻底停止，只能重启进程 | 暴露 `reconnect()` 方法，或通过 API 手动触发重连 |
| 6 | handler 丢弃未知事件 | Gateway 新增事件时无感知 | 未知事件仍触发通用 emitEvent，前端可自行监听 |

### 4.2 config 同步器修正

当前 `discoverFromConfig()` 的解析：

```typescript
// 错误：假设 config.agents 是数组
const config = JSON.parse(content) as { agents?: Record<string, unknown>[] }
if (!Array.isArray(config.agents)) return []
```

实际 `openclaw.json` 结构：

```json
{
  "agents": {
    "defaults": { "model": { "primary": "..." }, "workspace": "..." },
    "list": [
      { "id": "main", "name": "main", "workspace": "..." },
      { "id": "ceo", "name": "项目总监", "workspace": "..." }
    ]
  }
}
```

修正后应读取 `config.agents.list`，并将 `config.agents.defaults` 合并到每个 agent：

```typescript
interface OpenClawConfig {
  gateway?: { port?: number; auth?: { mode?: string; token?: string; password?: string } }
  agents?: {
    defaults?: { model?: unknown; workspace?: string }
    list?: Array<{ id: string; name?: string; workspace?: string; [k: string]: unknown }>
  }
}

// 解析 agents
const agents = config.agents?.list ?? []
const defaults = config.agents?.defaults ?? {}

return agents.map((a) => ({
  id: a.id,
  name: a.name ?? a.id,
  workspace: a.workspace ?? defaults.workspace ?? null,
  model: resolveModel(a.model ?? defaults.model),
  ...
}))
```
