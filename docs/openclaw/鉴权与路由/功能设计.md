# 鉴权与路由 功能设计

文件：`src/middleware/auth.ts`、`src/app.ts`、`src/routes/*.ts`

## 现状

- 两层鉴权机制（API Key + Service Token）
- 三类路由（公开 / 内部 / 开放）
- Request ID 追踪
- CORS + 全局错误处理

---

## 一、鉴权机制

### 1.1 两层设计

| 层 | 路由前缀 | 鉴权方式 | 使用方 |
|----|---------|---------|--------|
| 内部 | `/api/*` | API Key（Bearer Token） | Console 前端 |
| 开放 | `/open/*` | Service Token（X-Service-Token） | 业务服务（server） |
| 公开 | `/health`、`/api/gateway/status`、`/api/events/stream` | 无鉴权 | 任何客户端 |

### 1.2 API Key 鉴权

```
请求头：Authorization: Bearer {API_KEY}
校验：精确匹配 env.API_KEY
失败：401 Unauthorized
```

用于 Console 前端所有操作（CRUD、同步、活动日志等）。

### 1.3 Service Token 鉴权

```
请求头：X-Service-Token: {SERVICE_TOKEN}
校验：精确匹配 env.SERVICE_TOKEN
失败：401 Unauthorized
```

用于业务服务（server）调用 openclaw 能力。开放 API 权限受限：

- Agent / Skill：只读（GET）
- Task：读 + 创建（POST source=system）
- Session：读 + 创建（POST source=system）

## 二、路由结构

### 2.1 中间件栈

```
请求入站
  ↓
CORS（允许 CORS_ORIGIN）
  ↓
Hono Logger（请求日志）
  ↓
Request ID（X-Request-Id）
  ↓
路由匹配
  ├─ /health            → 无鉴权 → systemController.health
  ├─ /api/gateway/status → 无鉴权 → systemController.gatewayStatus
  ├─ /api/events/stream  → 无鉴权 → handleSSE
  ├─ /api/*              → API Key 鉴权 → 内部路由
  └─ /open/*             → Service Token 鉴权 → 开放路由
  ↓
全局错误处理
  ├─ ZodError → 400 BAD_REQUEST
  ├─ AppError → 对应 HTTP 状态码
  └─ 其他 → 500 INTERNAL_ERROR
```

### 2.2 内部路由（/api）

| 模块 | 挂载点 | 能力 |
|------|--------|------|
| Agents | `/api/agents` | 完整 CRUD + Soul + Logs + Sync |
| Skills | `/api/skills` | 完整 CRUD |
| Tasks | `/api/tasks` | 完整 CRUD |
| Sessions | `/api/sessions` | 完整 CRUD + Close |
| System | `/api/activities` | 活动日志查询 |

### 2.3 开放路由（/open）

| 模块 | 挂载点 | 能力 |
|------|--------|------|
| Agents | `/open/agents` | 只读（list + getById） |
| Skills | `/open/skills` | 只读（list + getById） |
| Tasks | `/open/tasks` | 读 + 创建（createFromOpen, source=system） |
| Sessions | `/open/sessions` | 读 + 创建（createFromOpen, source=system） |

## 三、统一响应格式

### 3.1 成功

```json
{
  "ok": true,
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 3.2 失败

```json
{
  "ok": false,
  "code": 10001,
  "message": "Agent not found",
  "data": null
}
```

### 3.3 错误处理

| 异常类型 | HTTP 状态 | 处理方式 |
|----------|----------|---------|
| ZodError | 400 | 提取 message，code=BAD_REQUEST |
| AppError | 自定义 | 使用 error.status / error.code / error.message |
| 未知异常 | 500 | 日志记录，返回通用错误信息 |

---

## 四、Request ID

每个请求自动分配唯一 ID（中间件注入）：

- 存储在 Hono Context Variables 的 `reqId` 中
- 可用于日志追踪和跨服务关联
- 响应头暂未返回（可扩展）
