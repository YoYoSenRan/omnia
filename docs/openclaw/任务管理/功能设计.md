# 任务管理 功能设计

文件：`src/services/task.ts`、`src/db/schema/task.ts`、`src/validators/task.ts`

## 现状

- Task CRUD 完整实现
- 状态机校验（7 状态 + 合法流转规则）
- 创建时若指定 assignedTo 则自动设为 assigned
- 完成/失败时自动记录 completedAt
- 事件触发 + 审计日志
- 内部 API 和开放 API 均可创建任务

## 一、状态机

### 1.1 状态定义

| 状态 | 含义 |
|------|------|
| `inbox` | 新建待处理，未分配 |
| `assigned` | 已分配给某个 Agent，等待执行 |
| `in_progress` | Agent 正在执行 |
| `review` | 执行完毕，等待评审 |
| `done` | 已完成 |
| `failed` | 执行失败 |
| `cancelled` | 已取消 |

### 1.2 状态流转

```
inbox ──→ assigned ──→ in_progress ──→ review ──→ done
  │          │             │              │
  │          │             ├──→ done      ├──→ in_progress（打回重做）
  │          │             ├──→ failed    └──→ failed
  │          │             └──→ cancelled
  │          └──→ cancelled
  └──→ cancelled

failed ──→ inbox（重新入列）
cancelled ──→ inbox（重新入列）
done ──→（终态，不可变更）
```

合法流转表：

```
inbox       → [assigned, cancelled]
assigned    → [in_progress, cancelled]
in_progress → [review, done, failed, cancelled]
review      → [done, in_progress, failed]
done        → []
failed      → [inbox]
cancelled   → [inbox]
```

非法流转时抛出 `TASK_STATUS_INVALID`（30002）。

### 1.3 自动行为

| 触发条件 | 自动行为 |
|----------|---------|
| 创建时指定 `assignedTo` | status 设为 `assigned`（而非 inbox） |
| status 变为 `done` / `failed` | 自动设置 `completedAt` |
| status 变为 `done` | 触发 `task.completed` 事件 |
| status 变更（非 done） | 触发 `task.updated` 事件 |

## 二、CRUD

### 2.1 创建

```
POST /api/tasks
Body: {
  title: string       (必填，min 1)
  description?: string
  priority?: number   (1–5，默认 3)
  assignedTo?: string (Agent ID)
  parentId?: string   (父任务 ID)
  tags?: string[]
}
```

- ID 自动生成（可选传入自定义 ID）
- source="user"（内部 API）或 source="system"（开放 API）

### 2.2 更新

```
PUT /api/tasks/:id
Body: {
  title?: string
  description?: string
  status?: TaskStatus
  priority?: number
  assignedTo?: string
  result?: string
  tags?: string[]
}
```

- status 变更会经过状态机校验
- 支持部分更新

### 2.3 快捷方法

| 方法 | 等价操作 |
|------|---------|
| `taskService.assign(taskId, agentId)` | update(id, { assignedTo, status: "assigned" }) |
| `taskService.complete(taskId, result?)` | update(id, { status: "done", result }) |
| `taskService.fail(taskId, error?)` | update(id, { status: "failed", result: error }) |

## 三、校验规则

### TaskCreateSchema (Zod)

| 字段 | 类型 | 约束 |
|------|------|------|
| id | string | 可选 |
| title | string | min(1) |
| description | string | nullish |
| priority | number | 1–5，可选 |
| assignedTo | string | nullish |
| parentId | string | 可选 |
| tags | string[] | nullish |

### TaskUpdateSchema (Zod)

所有字段均为 optional，status 限定为 TaskStatus 枚举值。
