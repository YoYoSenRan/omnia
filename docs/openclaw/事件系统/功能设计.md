# 事件系统 功能设计

文件：`src/events/bus.ts`、`src/events/sse.ts`

## 现状

- EventBus 基于 Node.js EventEmitter
- SSE 推送 8 种业务事件
- Service 层操作自动触发事件
- Gateway 消息也经由 EventBus 分发
- 客户端连接/断开自动管理

---

## 一、事件总线

### 1.1 架构

```
Service 操作 ──→ emitEvent() ──→ EventBus (EventEmitter)
                                     │
Gateway 消息 ──→ handler.ts ──→──────┤
                                     │
                                     ├──→ SSE 推送（客户端浏览器）
                                     └──→ 内部消费者（未来可扩展）
```

### 1.2 API

```typescript
// 发布事件
emitEvent(event: string, payload: Record<string, unknown>): void
// payload 自动追加 timestamp（ISO 8601）

// 订阅事件
eventBus.on(event, handler)
eventBus.off(event, handler)
```

### 1.3 命名规范

格式：`{module}.{action}`

| 事件 | 触发时机 | payload |
|------|---------|---------|
| `agent.synced` | 同步完成 | `{ created[], updated[], offlined[] }` |
| `agent.status` | Agent 状态变更 | `{ agentId, status }` |
| `task.created` | 任务创建 | `{ taskId, title }` |
| `task.updated` | 任务状态变更 | `{ taskId, status }` |
| `task.completed` | 任务完成 | `{ taskId }` |
| `session.created` | 会话创建 | `{ sessionId, agentId }` |
| `session.message` | 会话消息 | `{ sessionId, ... }` |
| `connection` | SSE 客户端首次连接 | `{ message: "connected" }` |

## 二、SSE 推送

### 2.1 端点

```
GET /api/events/stream
```

无鉴权（浏览器 EventSource 不支持自定义 Header）。

### 2.2 实现

使用 Hono `streamSSE()` API：

1. 客户端建立连接
2. 立即推送 `connected` 事件
3. 订阅 EventBus 上的所有业务事件
4. 收到事件后以 SSE 格式推送
5. 客户端断开时自动移除监听器

### 2.3 SSE 帧格式

```
event: agent.status
data: {"agentId":"abc","status":"idle","timestamp":"2026-03-13T10:00:00Z"}

event: task.created
data: {"taskId":"xyz","title":"Review PR","timestamp":"2026-03-13T10:01:00Z"}
```

### 2.4 订阅的事件列表

```typescript
[
  "connection",
  "agent.synced",
  "agent.status",
  "task.created",
  "task.updated",
  "task.completed",
  "session.created",
  "session.message",
]
```

## 三、事件来源

| 来源 | 触发方式 |
|------|---------|
| Service 层 | `emitEvent()` 在 create/update/remove 后调用 |
| Gateway handler | 收到网关 event 帧后转发到 EventBus |
| SyncService | 同步完成后触发 `agent.synced` |

### 3.1 Service 层触发点

| Service | 方法 | 事件 |
|---------|------|------|
| AgentService | afterCreate() | `agent.status` |
| AgentService | afterUpdate()（状态变更时） | `agent.status` |
| AgentService | updateStatus() | `agent.status` |
| TaskService | afterCreate() | `task.created` |
| TaskService | update()（status→done） | `task.completed` |
| TaskService | update()（其他状态变更） | `task.updated` |
| SessionService | afterCreate() | `session.created` |
| SyncService | syncAgents() | `agent.synced` |

## 四、Console 前端集成

Console 通过 `useSSE()` hook 订阅事件流，收到事件后 invalidate 对应的 TanStack Query 缓存：

```
agent.synced  → 刷新 agents + activities
agent.status  → 刷新 agents + activities
task.created  → 刷新 tasks + activities
task.updated  → 刷新 tasks + activities
task.completed → 刷新 tasks + activities
session.created → 刷新 sessions + activities
```
