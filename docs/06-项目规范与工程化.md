```

│   ├── .env.example

│   ├── .env.example
```

## 数据库连接

### 连接管理规范

每个后端服务统一使用以下模式管理数据库连接：

```ts
// db/index.ts
import { drizzle } from 'drizzle-orm/node-postgres'
import { Pool } from 'pg'
import * as schema from './schema'

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
})

export const db = drizzle(pool, { schema })
```

### 迁移管理

使用 drizzle-kit 管理数据库迁移：

```bash
# 生成迁移文件
pnpm drizzle-kit generate

# 应用迁移
pnpm drizzle-kit migrate

# 查看数据库（Drizzle Studio）
pnpm drizzle-kit studio
```

每个后端服务有独立的 `drizzle.config.ts`：

```ts
// apps/openclaw/drizzle.config.ts
import { defineConfig } from 'drizzle-kit'

export default defineConfig({
  schema: './src/db/schema.ts',
  out: './drizzle',
  dialect: 'postgresql',
  dbCredentials: {
    url: process.env.DATABASE_URL!,
  },
})
```

## 目录与文件命名

- 目录名：kebab-case（`core-client/`、`agent-sync.ts`）
- 组件文件：PascalCase（`AgentDetail.tsx`）
- 其他文件：kebab-case（`event-bus.ts`）
- 类型文件：kebab-case，不加 `.type` 后缀

## 统一响应格式

所有后端 API 统一响应结构：

```ts
// 成功
{ ok: true, code: 0, msg: 'success', data: T }

// 失败
{ ok: false, code: 'ERROR_CODE', msg: 'error message', data: null }
```

工具函数：

```ts
export function ok<T>(c: Context, data: T, status = 200) {
  return c.json({ ok: true, code: 0, msg: 'success', data }, status)
}

export function fail(c: Context, status: number, code: string, msg: string) {
  return c.json({ ok: false, code, msg, data: null }, status)
}
```

## 错误处理

- 后端：Hono 全局 `onError` 处理未捕获异常
- 前端：TanStack Query 的 `onError` 回调处理 API 错误
- 不添加掩盖错误的 fallback 机制，快速失败

## 脚本规范

根目录 `package.json` 提供统一入口：

```json
{
  "scripts": {
    "dev": "并行启动所有应用",
    "dev:core": "并行启动 openclaw + console",
    "dev:app": "并行启动 server + web",
    "build": "构建所有应用",
    "lint": "检查所有应用",
    "typecheck": "类型检查所有应用"
  }
}
```

每个应用内部也有独立的 `dev`、`build`、`lint` 脚本。
