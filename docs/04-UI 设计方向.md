# UI 设计方向

## 1. 设计目标

Omnia 的 UI 追求**专业感与视觉冲击力并存**。作为 AI Agent 控制台，界面需要传递"智能、精密、未来感"的气质，同时保持数据密集型场景下的可读性和操作效率。

对标产品视觉风格：

- **Linear** — 极致简洁、动效克制但精致
- **Raycast** — 深色主题、高对比度、响应迅速
- **Vercel Dashboard** — 数据密集但不杂乱、字体排版讲究
- **Arc Browser** — 大胆配色、活泼但不失专业

## 2. 主题与配色

### 2.1 默认深色主题

```
背景层级：
  - 最底层背景：   #09090B (zinc-950)
  - 卡片/面板背景： #18181B (zinc-900)
  - 悬浮/弹窗背景： #27272A (zinc-800)

文字层级：
  - 主文字：        #FAFAFA (zinc-50)
  - 次要文字：      #A1A1AA (zinc-400)
  - 禁用/提示文字：  #52525B (zinc-600)

主强调色：
  - 主色：          #6366F1 (indigo-500) — 按钮、选中态、关键操作
  - 主色悬浮：      #818CF8 (indigo-400)
  - 主色按下：      #4F46E5 (indigo-600)

状态色：
  - 成功/运行中：   #22C55E (green-500)
  - 警告：          #F59E0B (amber-500)
  - 错误：          #EF4444 (red-500)
  - 信息：          #3B82F6 (blue-500)

Agent 标识色（用于区分不同 Agent）：
  - 渐变色组：indigo / violet / cyan / emerald / amber / rose
  - 每个 Agent 分配独立色调，在事件流和 Chat 中一眼可辨
```

### 2.2 浅色主题

作为可选项支持，但深色为默认和主推。

## 3. 布局结构

### 3.1 整体布局

```
┌──────────────────────────────────────────────────────┐
│  顶部栏：Logo + 搜索 + 连接状态 + 设置              │
├────────┬─────────────────────────────────────────────┤
│        │                                             │
│  侧边  │              主内容区                       │
│  导航  │                                             │
│        │                                             │
│  图标  │  ┌──────────────────────────────────────┐  │
│  +     │  │  页面标题 + 操作按钮                  │  │
│  文字  │  ├──────────────────────────────────────┤  │
│        │  │                                      │  │
│  可    │  │  内容区域                             │  │
│  折    │  │  （列表/详情/编辑器/Chat）            │  │
│  叠    │  │                                      │  │
│        │  │                                      │  │
│        │  └──────────────────────────────────────┘  │
│        │                                             │
├────────┴─────────────────────────────────────────────┤
│  底部状态栏：Gateway 状态 + 活跃 Agent 数 + 版本     │
└──────────────────────────────────────────────────────┘
```

### 3.2 侧边导航

- 默认折叠为图标模式（64px 宽）
- 悬浮展开为图标 + 文字模式（240px 宽）
- 导航项：Dashboard / Agents / Skills / Workspace / Sessions / Chat / Executions / Projects / Models / Cron / Settings
- 每个导航项有独立图标（Lucide）
- 当前选中项有左侧高亮条 + 背景色变化

### 3.3 响应式

- 最小宽度 1024px（桌面端控制台，不考虑移动适配）
- 侧边栏在小屏幕下自动折叠
- 内容区自适应宽度

## 4. 关键页面设计方向

### 4.1 Dashboard

- **Bento Grid 布局**：不等宽卡片网格，大卡片放核心指标，小卡片放快捷操作
- **连接状态**：顶部醒目的状态条，连接中有呼吸动画，断连有红色脉冲
- **Agent 活动卡片**：每个活跃 Agent 一张卡片，显示实时状态，Agent 色调区分
- **事件流**：右侧实时滚动的事件流，新事件从底部滑入，有微妙的发光效果
- **数据图表**：最近 7 天的执行量趋势、Token 消耗统计

### 4.2 Agent 管理

- **卡片网格 + 列表切换**：默认卡片视图，每张卡片有 Agent emoji 和色调
- **Agent 详情**：左侧固定信息面板，右侧 Tab 切换内容区
- **状态指示**：实时脉冲点（绿色 = 运行中，灰色 = 空闲，红色 = 错误）

### 4.3 Chat 界面

- **全高布局**：充分利用垂直空间
- **消息气泡**：用户消息右对齐暗色背景，Agent 消息左对齐无背景
- **流式渲染**：Agent 回复逐字出现，有打字光标动画
- **工具调用块**：半透明背景卡片，可折叠，展示工具名称、参数和结果
- **思考过程**：斜体灰色文字，默认折叠
- **代码块**：语法高亮 + 一键复制按钮

### 4.4 执行追踪

- **时间线视图**：垂直时间线，每个节点是一个事件
- **事件节点**：不同事件类型用不同图标和颜色
- **展开详情**：点击节点展开详细信息面板
- **实时追踪**：新事件从底部追加，自动滚动，可手动暂停滚动

### 4.5 Workspace 编辑器

- **双面板布局**：左侧文件树，右侧编辑器
- **Monaco Editor**：深色主题（与整体风格一致）
- **Tab 切换**：多文件同时编辑
- **Markdown 预览**：分屏预览或切换预览

### 4.6 项目模板

- **模板画廊**：大卡片展示，有封面色/图标/描述
- **创建向导**：分步表单，有 Agent 关系的可视化预览
- **执行看板**：运行中的实例以看板形式展示进度

## 5. 动效设计

### 5.1 原则

- 动效服务于反馈和引导，不做纯装饰性动画
- 时长控制在 150-300ms
- 使用 ease-out 曲线为主

### 5.2 关键动效

| 场景 | 动效 |
|------|------|
| 页面切换 | 内容区淡入 + 微小上移（150ms） |
| 卡片悬浮 | 微妙上移 2px + 阴影增强 |
| 侧边栏展开 | 宽度过渡 200ms |
| 事件流新增 | 底部滑入 + 短暂高亮背景 |
| Chat 消息 | 淡入 + 从下方微小滑入 |
| 流式文字 | 光标闪烁动画 |
| 连接状态 | 状态点呼吸脉冲动画 |
| 审批弹窗 | 从底部滑入 + 背景模糊 |
| Toast 通知 | 从右上角滑入，3s 后自动滑出 |
| 加载状态 | 骨架屏（Skeleton）渐变闪烁 |
| 数据图表 | 首次加载时数据点逐个入场 |

### 5.3 微交互

- 按钮按下有轻微缩放反馈（scale 0.98）
- 输入框聚焦时边框渐变到主色
- 复制按钮点击后图标切换为对勾（300ms 后恢复）
- 切换开关有弹性过渡

## 6. 视觉特征

### 6.1 玻璃拟态（适度使用）

- 顶部栏和侧边栏使用半透明 + backdrop-blur
- 弹窗和浮层使用磨砂玻璃背景
- 不滥用——只在浮动层级使用，卡片和内容区用实色

### 6.2 边框与分隔

- 使用极细的 1px 边框（border-zinc-800）
- 分隔线用 zinc-800/50（半透明）
- 避免过多嵌套边框

### 6.3 圆角

- 卡片和按钮：8px（rounded-lg）
- 输入框：6px（rounded-md）
- 大容器/面板：12px（rounded-xl）
- 头像/图标：圆形

### 6.4 阴影

- 深色主题下阴影效果弱，改用边框和背景色差异区分层级
- 弹窗使用柔和阴影 + 边框组合
- 卡片悬浮时使用主色调的微弱 glow 效果

## 7. 字体

```css
/* 主字体：系统默认 */
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;

/* 代码字体 */
font-family: 'JetBrains Mono', 'Fira Code', 'SF Mono', monospace;
```

字号层级：

| 用途 | 大小 |
|------|------|
| 页面标题 | 24px / font-semibold |
| 区块标题 | 18px / font-medium |
| 正文 | 14px / font-normal |
| 辅助文字 | 12px / font-normal |
| 代码/数据 | 13px / monospace |

## 8. 图标与标识

- 使用 Lucide React 图标库
- Agent 使用 emoji 作为标识（与 OpenClaw 一致）
- 状态使用彩色圆点（8px）
- 事件类型使用特定图标：
  - 消息：MessageSquare
  - 工具调用：Wrench
  - 错误：AlertCircle
  - 审批：ShieldCheck
  - 完成：CheckCircle
