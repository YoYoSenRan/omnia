# 功能模块设计

## 1. 模块总览

Omnia 的功能围绕 OpenClaw 的核心概念展开，分为以下模块：

| 模块 | 说明 |
|------|------|
| Dashboard | 系统总览、连接状态、Agent 活动概览 |
| Agent 管理 | Agent CRUD、配置编辑、状态监控 |
| Skill 管理 | Skill 浏览、安装、更新、配置查看 |
| Workspace 管理 | 文件浏览与编辑、配置文件管理 |
| Session 管理 | Session 列表、历史查看、重置/压缩 |
| Chat | 与 Agent 实时对话、工具调用展示 |
| 执行追踪 | 实时事件流、执行历史、产物查看 |
| 项目模板 | 多 Agent 组合定义、一键启动 |
| Model 管理 | 可用模型查看、Agent 模型配置 |
| Cron 管理 | 定时任务查看、添加、手动触发 |
| 系统设置 | Gateway 连接配置、UI 偏好 |

## 2. Dashboard

### 2.1 功能描述

系统首页，一屏展示 Omnia 与 OpenClaw 的整体状态。

### 2.2 展示内容

- **连接状态**：Gateway 连接状态指示灯（connected / connecting / disconnected）
- **Agent 概览**：已配置 Agent 数量、当前活跃 Agent、最近活动
- **Session 概览**：活跃 Session 数量、最近交互时间
- **执行动态**：最近的执行事件流（滚动展示）
- **系统健康**：Gateway 运行时长、内存占用、模型提供商状态

### 2.3 交互

- 点击任意卡片可跳转到对应详情页
- 执行动态支持实时更新（SSE 推送）

## 3. Agent 管理

### 3.1 功能描述

管理 OpenClaw 中的所有 Agent，包括查看、创建、配置和删除。

### 3.2 Agent 列表

展示所有 Agent，每个 Agent 卡片包含：

- Agent 名称 + emoji 标识
- 绑定的 Workspace 路径
- 使用的模型
- 当前状态（idle / running / error）
- 最近活动时间
- 关联的 Channel 数量

### 3.3 Agent 详情

点击 Agent 进入详情页，包含以下 Tab：

**概览 Tab**
- Agent 基本信息
- 运行状态实时指示
- 最近执行摘要

**配置 Tab**
- Workspace 文件编辑器（AGENTS.md / SOUL.md / IDENTITY.md / USER.md / TOOLS.md）
- 支持 Markdown 预览和代码编辑双模式
- 模型选择下拉框
- Sandbox 配置
- 工具策略配置（allow / deny 列表）

**Skills Tab**
- 该 Agent 已加载的 Skill 列表
- 快捷安装/移除入口

**Sessions Tab**
- 该 Agent 关联的所有 Session
- 支持查看、重置、压缩

**执行历史 Tab**
- 该 Agent 的历史执行记录
- 按时间倒序排列
- 支持展开查看执行详情

**记忆 Tab**
- 查看 MEMORY.md 内容
- 查看最近的 daily logs
- 记忆条目搜索（调用 OpenClaw memory_search）

### 3.4 创建 Agent

- 表单输入：名称、emoji、模型选择
- 可选择从现有 Agent 克隆配置
- 可选择从项目模板创建
- 自动生成 Workspace 目录和初始配置文件

### 3.5 多 Agent 路由视图

可视化展示 Agent 与 Channel 的路由映射关系（来自 openclaw.json 的 agents.mapping 配置），支持直观的拖拽配置。

## 4. Skill 管理

### 4.1 功能描述

浏览和管理 OpenClaw 已安装的 Skill。

### 4.2 Skill 列表

展示所有 Skill，每个 Skill 卡片包含：

- Skill 名称 + 图标
- 简短描述
- 来源（bundled / managed / workspace override）
- 版本号
- 关联的 Agent 列表

### 4.3 Skill 详情

- SKILL.md 内容展示（Markdown 渲染）
- 使用说明与示例
- 配置参数（如有）
- 安全风险提示

### 4.4 Skill 安装

- 输入 Skill URL 或从推荐列表选择
- 调用 `skills.install` 安装
- 安装后自动刷新列表

### 4.5 Skill 更新

- 检测可更新的 Skill
- 一键更新或批量更新

## 5. Workspace 管理

### 5.1 功能描述

管理 OpenClaw Workspace 文件系统，提供可视化的文件浏览和编辑能力。

### 5.2 文件浏览器

- 树形展示 Workspace 目录结构
- 区分文件类型图标（Markdown / JSON / YAML / 其他）
- 支持展开/折叠目录

### 5.3 文件编辑器

- 内置代码编辑器（Monaco Editor）
- Markdown 文件支持预览模式
- 语法高亮（Markdown / JSON / YAML / TypeScript）
- 保存时自动写回文件系统

### 5.4 关键文件快捷入口

为以下文件提供顶部 Tab 快捷切换：

- AGENTS.md
- SOUL.md
- USER.md
- IDENTITY.md
- TOOLS.md
- MEMORY.md
- openclaw.json

### 5.5 多 Workspace 支持

- 列出所有配置的 Workspace（包括 agents-workspaces/ 下的）
- Workspace 切换选择器

## 6. Session 管理

### 6.1 功能描述

查看和管理 OpenClaw 的所有 Session。

### 6.2 Session 列表

- 展示所有活跃 Session
- 每个 Session 显示：ID 格式、关联 Agent、关联 Channel、创建时间、消息数
- 按 Agent 分组视图 / 平铺视图切换

### 6.3 Session 详情

- 完整对话历史浏览
- 消息气泡展示（区分 user / assistant / tool）
- 工具调用折叠展示
- 操作按钮：重置、压缩、导出

### 6.4 Session 操作

| 操作 | 说明 |
|------|------|
| 重置 | 清空 Session 上下文，重新开始 |
| 压缩 | 压缩历史消息，保留摘要 |
| 导出 | 将对话历史导出为 Markdown / JSON |

## 7. Chat

### 7.1 功能描述

与 Agent 实时对话的核心交互界面。

### 7.2 Chat 界面

- Agent 选择器（顶部切换当前对话 Agent）
- 消息输入框（支持多行、Markdown）
- 消息流展示：
  - 用户消息
  - Agent 回复（流式渲染，逐 token 展示）
  - 工具调用块（可折叠，展示工具名、参数、结果）
  - 思考过程（如有，折叠展示）
- 操作按钮：中止执行、重新生成

### 7.3 审批弹窗

当 Agent 请求执行需要审批的操作时：

- 弹出审批面板
- 展示操作详情（命令内容、影响范围）
- 风险等级提示
- 批准 / 拒绝按钮

### 7.4 多 Session 支持

- 侧边栏展示当前 Agent 的所有 Session
- 支持创建新 Session
- 支持在 Session 间切换

## 8. 执行追踪

### 8.1 功能描述

实时追踪和回溯 Agent 的执行过程。

### 8.2 实时事件流

- 全局事件流面板（类似日志查看器）
- 按时间线展示所有 Agent 的执行事件
- 事件类型标签（消息、工具调用、错误、审批）
- 支持按 Agent / 事件类型过滤

### 8.3 执行详情

每次执行（一轮完整的 send → response）记录：

- 触发时间
- 关联 Agent 和 Session
- 消息内容
- 工具调用链（按顺序展示每个工具的调用参数和结果）
- Token 消耗
- 执行时长
- 最终状态（完成 / 错误 / 中止）

### 8.4 执行历史

- 按时间倒序列出历史执行记录
- 支持搜索和过滤
- 点击可展开完整执行轨迹

## 9. 项目模板

### 9.1 功能描述

定义多 Agent 协作的可复用模板，一键启动特定领域的工作流。

### 9.2 模板定义

一个项目模板包含：

```yaml
name: "技术调研"
description: "自动搜索、分析、产出调研报告"
agents:
  - role: researcher
    soul: "你是一个技术研究员，擅长在互联网上搜索和整理技术信息..."
    skills: [web-search, browser]
    model: anthropic/claude-sonnet-4-5
  - role: analyst
    soul: "你是一个技术分析师，擅长从原始信息中提炼要点..."
    skills: [file-write]
    model: anthropic/claude-sonnet-4-5
workflow:
  - step: "信息采集"
    agent: researcher
    prompt: "搜索并整理关于 {{topic}} 的最新技术动态"
  - step: "分析产出"
    agent: analyst
    prompt: "基于以下信息，产出一份结构化调研报告：{{previous_output}}"
```

### 9.3 模板管理

- 模板列表展示
- 模板创建/编辑（表单 + YAML 编辑双模式）
- 模板导入/导出

### 9.4 项目启动

- 选择模板
- 填入变量（如 topic）
- 一键启动：自动创建 Agent Workspace、发送初始指令
- 展示执行进度

### 9.5 项目实例

- 记录每次从模板启动的项目实例
- 展示实例状态、执行进度、产出结果
- 支持中止和重启

## 10. Model 管理

### 10.1 功能描述

查看和管理可用的 LLM 模型。

### 10.2 功能

- 调用 `models.list` 展示所有可用模型
- 按提供商分组（Anthropic / OpenAI / Google / Local）
- 展示模型能力标签（vision / tool-use / reasoning）
- 在 Agent 配置中可快速选择模型

## 11. Cron 管理

### 11.1 功能描述

管理 OpenClaw 的定时任务。

### 11.2 功能

- 展示所有 Cron 任务
- 创建新 Cron 任务（cron 表达式 + 目标 Agent + 执行内容）
- 手动触发执行
- 查看执行历史
- 启用/停用任务

## 12. 系统设置

### 12.1 Gateway 连接

- Gateway 地址配置（默认 ws://localhost:18789）
- 认证 Token 配置
- 连接状态和诊断信息

### 12.2 UI 偏好

- 主题设置（深色 / 浅色，默认深色）
- 语言设置
- 布局偏好
- 通知设置
