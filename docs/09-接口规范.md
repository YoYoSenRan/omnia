# 09 - 接口规范

## 统一响应格式

所有 API 接口使用统一的 JSON 响应结构：

```ts
interface ApiResponse<T = unknown> {
  /** 请求是否成功 */
  ok: boolean
  /** 业务状态码，0 表示成功，非 0 为具体错误码 */
  code: number
  /** 人类可读的消息，成功时为 "success"，失败时为错误描述 */
  message: string
  /** 响应数据，失败时为 null */
  data: T | null
}
```

### 成功响应

```json
{
  "ok": true,
  "code": 0,
  "message": "success",
  "data": {
    "id": "agent-001",
    "name": "Researcher"
  }
}
```

### 失败响应

```json
{
  "ok": false,
  "code": 40001,
  "message": "Agent not found",
  "data": null
}
```

### 列表响应（带分页）

```json
{
  "ok": true,
  "code": 0,
  "message": "success",
  "data": {
    "items": [...],
    "total": 42,
    "page": 1,
    "pageSize": 20
  }
}
```

## HTTP 状态码

HTTP 状态码表示传输层结果，业务错误通过 `code` 字段区分。

| HTTP 状态码 | 含义 | 使用场景 |
|------------|------|---------|
| 200 | OK | 查询、更新、删除成功 |
| 201 | Created | 创建资源成功 |
| 400 | Bad Request | 请求参数校验失败 |
| 401 | Unauthorized | 未认证或 token 无效 |
| 403 | Forbidden | 无权限访问 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 资源冲突（如重复创建） |
| 422 | Unprocessable Entity | 参数格式正确但语义错误 |
| 500 | Internal Server Error | 服务端未捕获异常 |
| 503 | Service Unavailable | 依赖服务不可用（如网关离线） |

## 业务状态码（code）

### 编码规则

5 位数字，格式为 `XXYYZ`：

- `XX`：模块，`00`=通用，`10`=Agent，`20`=Skill，`30`=Task，`40`=Session，`50`=Gateway
- `YY`：错误序号
- `Z`：保留，默认 `0`

### 通用错误（00xxx）

| code | 含义 |
|------|------|
| 0 | 成功（唯一的成功码） |
| 400 | 请求参数无效 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务内部错误 |

### Agent 错误（10xxx）

| code | 含义 |
|------|------|
| 10001 | Agent 不存在 |
| 10002 | Agent 名称重复 |
| 10003 | Agent 同步失败 |
| 10004 | Agent 当前状态不允许此操作 |
| 10005 | Soul 内容读取失败 |
| 10006 | Soul 内容写入失败 |

### Skill 错误（20xxx）

| code | 含义 |
|------|------|
| 20001 | Skill 不存在 |
| 20002 | Skill 同步失败 |

### Task 错误（30xxx）

| code | 含义 |
|------|------|
| 30001 | Task 不存在 |
| 30002 | Task 状态流转非法 |
| 30003 | Task 分配失败，目标 Agent 不可用 |
| 30004 | Task 执行超时 |

### Session 错误（40xxx）

| code | 含义 |
|------|------|
| 40001 | Session 不存在 |
| 40002 | Session 已关闭 |

### Gateway 错误（50xxx）

| code | 含义 |
|------|------|
| 50001 | 网关连接失败 |
| 50002 | 网关未连接 |
| 50003 | 网关认证失败 |
| 50004 | 网关请求超时 |

## 响应工具函数

### 后端实现

```ts
/**
 * API 响应工具函数
 *
 * 提供统一的成功/失败响应构造方法，
 * 确保所有接口返回格式一致。
 *
 * @module response
 */

import type { Context } from 'hono'

/** 业务状态码常量 */
export const CODE = {
  /** 成功 */
  SUCCESS: 0,

  /** 通用错误 */
  BAD_REQUEST: 400,
  UNAUTHORIZED: 401,
  FORBIDDEN: 403,
  NOT_FOUND: 404,
  INTERNAL_ERROR: 500,

  /** Agent 相关 */
  AGENT_NOT_FOUND: 10001,
  AGENT_NAME_DUPLICATE: 10002,
  AGENT_SYNC_FAILED: 10003,
  AGENT_INVALID_STATE: 10004,
  AGENT_SOUL_READ_FAILED: 10005,
  AGENT_SOUL_WRITE_FAILED: 10006,

  /** Skill 相关 */
  SKILL_NOT_FOUND: 20001,
  SKILL_SYNC_FAILED: 20002,

  /** Task 相关 */
  TASK_NOT_FOUND: 30001,
  TASK_INVALID_TRANSITION: 30002,
  TASK_ASSIGN_FAILED: 30003,
  TASK_TIMEOUT: 30004,

  /** Session 相关 */
  SESSION_NOT_FOUND: 40001,
  SESSION_CLOSED: 40002,

  /** Gateway 相关 */
  GATEWAY_CONNECT_FAILED: 50001,
  GATEWAY_DISCONNECTED: 50002,
  GATEWAY_AUTH_FAILED: 50003,
  GATEWAY_TIMEOUT: 50004,
} as const

/**
 * 构造成功响应
 *
 * @param c - Hono 请求上下文
 * @param data - 响应数据
 * @param status - HTTP 状态码，默认 200
 */
export function ok<T>(c: Context, data: T, status: 200 | 201 = 200) {
  return c.json({ ok: true, code: 0, message: 'success', data }, status)
}

/**
 * 构造失败响应
 *
 * @param c - Hono 请求上下文
 * @param httpStatus - HTTP 状态码
 * @param code - 业务状态码
 * @param message - 错误描述
 */
export function fail(
  c: Context,
  httpStatus: number,
  code: number,
  message: string,
) {
  return c.json(
    { ok: false, code, message, data: null },
    httpStatus as any,
  )
}
```

### 使用示例

```ts
/** GET /api/agents/:id — 查询单个 Agent 详情 */
app.get('/:id', (c) => {
  const id = c.req.param('id')
  const agent = getAgent(id)

  if (!agent) {
    return fail(c, 404, CODE.AGENT_NOT_FOUND, 'Agent not found')
  }

  return ok(c, agent)
})

/** POST /api/agents — 创建新 Agent */
app.post('/', async (c) => {
  const body = await c.req.json()

  if (!body.name) {
    return fail(c, 400, CODE.BAD_REQUEST, 'name is required')
  }

  const agent = createAgent(body)
  return ok(c, agent, 201)
})
```

### 前端处理

```ts
/**
 * 统一请求函数
 *
 * 自动解析响应 JSON，检查 ok 字段，
 * 失败时抛出包含 code 和 message 的 ApiError。
 */
async function request<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(`${API_BASE}${path}`, {
    ...init,
    headers: {
      'Content-Type': 'application/json',
      ...init?.headers,
    },
  })

  const json = await res.json() as ApiResponse<T>

  if (!json.ok) {
    throw new ApiError(json.code, json.message)
  }

  return json.data as T
}
```

## 分页参数

列表接口统一使用 query 参数分页：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| page | number | 1 | 当前页码，从 1 开始 |
| pageSize | number | 20 | 每页条数，最大 100 |
| sort | string | created_at | 排序字段 |
| order | string | desc | 排序方向：asc / desc |

请求示例：

```
GET /api/agents?page=1&pageSize=20&sort=updated_at&order=desc
```

响应：

```json
{
  "ok": true,
  "code": 0,
  "message": "success",
  "data": {
    "items": [...],
    "total": 42,
    "page": 1,
    "pageSize": 20
  }
}
```

## SSE 事件格式

SSE 推送的事件使用以下格式：

```
event: agent.status
data: {"id":"agent-001","status":"running","timestamp":"2026-03-13T10:00:00Z"}

event: task.updated
data: {"id":"task-001","status":"done","result":"...","timestamp":"2026-03-13T10:00:05Z"}
```

事件命名规则：`{模块}.{动作}`，如 `agent.synced`、`task.created`、`session.message`。
