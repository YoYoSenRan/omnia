# 07 - 实施阶段

## Phase 1：项目骨架

搭建 monorepo 基础结构，四个应用能独立启动。

### 任务

- [ ] 初始化 pnpm workspace
- [ ] 创建 apps/openclaw 骨架（Hono + PG + Drizzle）
- [ ] 创建 apps/console 骨架（React + Vite + Tailwind + shadcn）
- [ ] 创建 apps/server 骨架（Hono + PG + Drizzle）
- [ ] 创建 apps/web 骨架（React + Vite + Tailwind + shadcn）
- [ ] 配置环境变量模板（.env.example）
- [ ] 配置根目录统一脚本
- [ ] 验证四个应用能各自独立启动

### 交付标准

- `pnpm dev:core` 启动 openclaw(3001) + console(5173)
- `pnpm dev:app` 启动 server(3002) + web(5174)
- 各服务返回基本的健康检查响应

---

## Phase 2：OpenClaw 对接服务（openclaw）

核心能力建设，实现网关连接和数据同步。

### 任务

- [ ] 实现 OpenClaw WebSocket 客户端（从 omnia 迁移适配）
- [ ] 实现 Agent 多源同步（网关 + 磁盘 + 配置文件）
- [ ] 实现 Skill 同步
- [ ] 建立数据库 schema（agents、skills、tasks、sessions、audit_logs）
- [ ] 实现内部 API（全功能 CRUD）
- [ ] 实现开放 API（查询 + 调度）
- [ ] 实现事件总线 + SSE 推送
- [ ] 实现审计日志

### 交付标准

- openclaw 服务能连接 OpenClaw Gateway
- Agent / Skill 数据能同步到 PostgreSQL
- API 能正常返回 agent 列表、提交任务
- SSE 能推送实时事件

---

## Phase 3：控制台前端（console）

可视化管理界面。

### 任务

- [ ] 实现布局（Sidebar + Header + 主内容区）
- [ ] Dashboard 页面（网关状态、统计数据）
- [ ] Agent 列表页（状态标签、来源标签、同步按钮）
- [ ] Agent 详情页（信息编辑 + Soul 编辑 + 审计日志）
- [ ] Skill 列表页
- [ ] Session 列表页
- [ ] Task 看板页
- [ ] Settings 页面（网关连接配置）

### 交付标准

- 控制台能展示 openclaw 服务的所有数据
- Agent CRUD 操作正常

- 实时状态更新正常

---

## Phase 4：业务后端基础（server）

业务服务框架和 Core 客户端。

### 任务

- [ ] 实现 Core 客户端（封装对 openclaw 的 HTTP 调用）
- [ ] 实现 Core 事件订阅（SSE 客户端）
- [ ] 建立业务数据库 schema
- [ ] 实现基础业务 API 框架
- [ ] 实现统一错误处理

### 交付标准

- server 能通过 Core 客户端获取 agent 列表
- server 能提交任务到 openclaw 并获取结果
- server 能接收 openclaw 的 SSE 事件

---

## Phase 5：业务前端基础（web）

首个垂直项目的 UI。

### 任务

- [ ] 实现布局框架
- [ ] 需求输入界面
- [ ] 任务流程展示
- [ ] Agent 运行状态展示
- [ ] 产出物展示

### 交付标准

- 用户能通过界面输入需求
- 能看到任务拆解和执行过程
- 能查看产出结果

---

## Phase 6：端到端打通

全链路验证。

### 任务

- [ ] 用户在 web 输入需求 → server 接收 → 调 openclaw 拆解 → 分配 agent → 执行 → 返回结果 → web 展示
- [ ] console 实时展示整个过程
- [ ] 异常场景处理（网关断线、agent 失败、超时）

### 交付标准

- 完整链路跑通
- 控制台和业务前端同时可用
- 异常情况有合理的错误提示
