# 11 - 服务间通信与鉴权

## 通信模式

```
console ──HTTP──→ openclaw     （前端 → Core，带 API Key）
web ──HTTP──→ server            （前端 → 业务，带 API Key）
server ──HTTP──→ openclaw       （服务间，带 Service Token）
openclaw ──SSE──→ server        （事件推送，带 Service Token）
openclaw ──WebSocket──→ Gateway （网关协议，带 Gateway Token）
```

## 鉴权设计

### 三种身份

| 身份 | Header | 用途 |
|------|--------|------|
| 前端用户 | `Authorization: Bearer <api-key>` | console/web 访问后端 |
| 内部服务 | `X-Service-Token: <token>` | server 调用 openclaw |
| 网关连接 | Gateway 协议内置认证 | openclaw 连接 OpenClaw Gateway |

### 环境变量配置

```env
# openclaw/.env
API_KEY=your-console-api-key        # console 访问 openclaw 的凭证
SERVICE_TOKEN=your-service-token    # server 访问 openclaw 的凭证

# server/.env
API_KEY=your-web-api-key            # web 访问 server 的凭证
SERVICE_TOKEN=your-service-token    # 与 openclaw 的 SERVICE_TOKEN 一致
```

### 鉴权中间件

```ts
/**
 * API Key 鉴权中间件
 *
 * 验证 Authorization header 中的 Bearer token。
 * 用于前端请求的身份验证。
 */
function apiKeyAuth(apiKey: string) {
  return async (c: Context, next: Next) => {
    const auth = c.req.header('Authorization')
    if (!auth || auth !== `Bearer ${apiKey}`) {
      return fail(c, 401, CODE.UNAUTHORIZED, 'Invalid API key')
    }
    return next()
  }
}

/**
 * 服务间鉴权中间件
 *
 * 验证 X-Service-Token header。
 * 用于 server → openclaw 的内部调用。
 */
function serviceAuth(serviceToken: string) {
  return async (c: Context, next: Next) => {
    const token = c.req.header('X-Service-Token')
    if (!token || token !== serviceToken) {
      return fail(c, 401, CODE.UNAUTHORIZED, 'Invalid service token')
    }
    return next()
  }
}
```

### openclaw 路由鉴权划分

```ts
// 内部 API — console 通过 API Key 访问
app.use('/api/*', apiKeyAuth(process.env.API_KEY))

// 开放 API — server 通过 Service Token 访问
app.use('/open/*', serviceAuth(process.env.SERVICE_TOKEN))

// 公开接口 — 无需鉴权
// GET /health
```

### 路径前缀约定

| 前缀 | 鉴权方式 | 访问者 |
|------|---------|-------|
| `/api/*` | API Key | console 前端 |
| `/open/*` | Service Token | server 服务 |
| `/health` | 无 | 任何人 |

## 超时与重试

### server 调用 openclaw

```ts
/** 默认请求超时（毫秒） */
const DEFAULT_TIMEOUT = 10_000

/** 长时间运行的任务超时（毫秒），如 agent 执行 */
const LONG_TASK_TIMEOUT = 120_000

/** 最大重试次数 */
const MAX_RETRIES = 3

/** 重试间隔基数（毫秒），实际间隔 = BASE * 2^(retry-1) */
const RETRY_BASE_DELAY = 1000
```

### 重试策略

- **可重试**：网络错误、超时、502/503/504
- **不可重试**：400、401、403、404、422
- **退避算法**：指数退避（1s → 2s → 4s）

### 降级策略

当 openclaw 不可用时，server 的行为：

| 接口类型 | 降级方式 |
|---------|---------|
| 查询类（GET） | 返回缓存数据或空列表 + 降级标记 |
| 写入类（POST/PUT） | 直接返回 503 错误，不静默失败 |
| 任务提交 | 返回 503，前端提示用户稍后重试 |

降级响应示例：

```json
{
  "ok": false,
  "code": 50002,
  "message": "Core service unavailable, please try again later",
  "data": null
}
```

## CORS 配置

### openclaw

```ts
app.use('*', cors({
  origin: [
    'http://localhost:5301',  // console 开发服务器
  ],
  credentials: true,
}))
```

### server

```ts
app.use('*', cors({
  origin: [
    'http://localhost:5302',  // web 开发服务器
  ],
  credentials: true,
}))
```

生产环境通过环境变量 `CORS_ORIGIN` 配置。
