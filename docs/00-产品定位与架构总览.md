# Omnia 产品定位与架构总览

## 1. 产品定位

Omnia 是一个基于 OpenClaw 的 Web 控制台，为个人用户提供多 Agent 项目协作的上层管理能力。

Omnia 不重做 Agent runtime，不重做 LLM 调用、工具执行、沙箱或安全互锁。Omnia 通过 WebSocket 连接已运行的 OpenClaw Gateway，在其之上提供 Agent 管理、Skill 管理、Workspace 配置、Session 管理、执行追踪和项目编排能力。

## 2. 目标用户

单个操作者，即 OpenClaw 实例的所有者本人。

典型特征：

- 已在本地运行 OpenClaw Gateway
- 需要管理多个 Agent 及其 Skill 配置
- 需要将多个 Agent 组合成项目，协作完成特定领域任务
- 需要追踪 Agent 执行过程和产出
- 希望通过持续使用积累记忆，让 Agent 越来越懂自己，逐步减少人工决策

## 3. 核心价值

### 3.1 可视化管控

将 OpenClaw 的 Agent、Skill、Workspace、Session 等概念以直观的可视化界面呈现，替代命令行和配置文件操作。

### 3.2 项目化协作

将多个 Agent 按项目组织，定义协作模板，一键启动多 Agent 工作流，而不是逐个手动配置和调用。

### 3.3 执行可追踪

实时展示 Agent 执行过程（工具调用、消息流、状态变化），执行完成后可回溯完整轨迹。

### 3.4 记忆可进化

基于 OpenClaw 原生记忆系统，未来逐步增强偏好提炼、经验积累和自主决策能力，让 Agent 越用越智能。

## 4. 非目标

当前阶段不解决：

- 多用户协作与团队管理
- SaaS 多租户与计费
- 重做 OpenClaw 的 Agent runtime / tool runtime / sandbox
- 重做 OpenClaw 的记忆系统（当前使用 OpenClaw 默认实现）
- 管理 OpenClaw Gateway 的启停（假设已在运行）

## 5. 系统边界

### 5.1 Omnia 负责

- Agent 可视化管理（CRUD、状态查看、配置编辑）
- Skill 管理（查看、安装、更新、配置）
- Workspace 管理（文件浏览、配置编辑、模板管理）
- Session 管理（查看、重置、压缩、历史浏览）
- 项目模板（定义 Agent 组合与工作流）
- 执行追踪（实时事件流、执行历史、产物收集）
- Chat 界面（与 Agent 直接对话）

### 5.2 OpenClaw 负责

- Agent runtime（模型推理、工具执行、沙箱）
- Gateway 协议（WebSocket 通信、认证、设备管理）
- 记忆系统（MEMORY.md、daily logs、向量检索）
- Skills 执行（SKILL.md 解析、工具调用）
- Session 状态（会话持久化、上下文压缩）
- 多 Agent 路由（channel mapping、workspace 隔离）
- Cron / Hooks / Webhooks

## 7. 数据职责划分

| 数据 | 归属 | 说明 |
|------|------|------|
| Agent 配置 | OpenClaw | AGENTS.md / openclaw.json |
| Skill 定义 | OpenClaw | skills/ 目录下的 SKILL.md |
| Workspace 文件 | OpenClaw | ~/.openclaw/workspace/ |
| Session 状态 | OpenClaw | 会话 JSON 文件 |
| 记忆 | OpenClaw | MEMORY.md + memory/ + SQLite 向量库 |
| 项目模板 | Omnia | SQLite 存储 |
| 执行历史 | Omnia | 事件流快照、执行摘要 |
| 用户偏好 | Omnia | UI 配置、布局偏好 |

## 8. 核心设计原则

### 8.1 不重复造轮子

OpenClaw 已有的能力直接通过 Gateway 协议调用，不在 Omnia 中重新实现。

### 8.2 Adapter 隔离协议细节

Omnia 业务层不直接依赖 Gateway 协议格式。所有 OpenClaw 通信收敛到 Adapter 层，向上暴露语义化接口。

### 8.3 渐进增强

先做 OpenClaw 能力的可视化壳，跑通基础管理和执行流程。记忆增强、自动化编排、智能决策作为后续迭代方向。

### 8.4 前端不持有业务真相

前端只负责展示和操作触发。Agent 状态、Session 状态等业务真相由 OpenClaw 持有，Omnia Server 作为代理层透传。
