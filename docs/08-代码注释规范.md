# 08 - 代码注释规范

## 总则

所有生成的代码必须包含清晰、标准的注释。注释是代码的一部分，不是可选项。

---

## JSDoc 注释

### 函数 / 方法

所有导出函数和关键内部函数必须使用 JSDoc 注释：

```ts
/**
 * 将网关返回的 Agent 数据与本地数据库记录合并
 *
 * 网关数据优先级高于本地数据，但本地独有字段（soul、config）会保留。
 * 当网关离线时返回纯本地数据。
 *
 * @param localAgents - 本地数据库中的 Agent 列表

 * @returns 合并后的完整 Agent 列表
 */
export function mergeAgents(
  localAgents: LocalAgent[],
  gatewayAgents: Agent[],
): LocalAgent[] {
  // ...
}
```

### 接口 / 类型

所有导出的接口和类型必须注释用途，字段需要行内注释：

```ts
/** 本地持久化的 Agent，在网关 Agent 基础上扩展了来源和同步信息 */
export interface LocalAgent extends Agent {
  /** 数据来源：gateway=网关同步 / local=磁盘发现 / config=配置文件 */
  source: AgentSource
  /** 来源引用，如连接 ID、磁盘路径或配置文件路径 */
  sourceRef?: string
  /** soul.md 内容缓存 */
  soul?: string
  /** 额外配置，JSON 格式 */
  config?: string
  /** 最后一次同步时间（ISO 8601） */
  lastSyncAt?: string
  /** 记录创建时间（ISO 8601） */
  createdAt: string
  /** 记录更新时间（ISO 8601） */
  updatedAt: string
}
```

### 常量 / 配置

```ts
/** SSE 重连间隔（毫秒） */
const SSE_RECONNECT_INTERVAL = 3000

/** 任务调度器轮询间隔（毫秒） */
```

## 逻辑注释

### 代码块内的步骤说明

复杂逻辑必须用注释标注每个步骤的意图：

```ts
async function syncAgents(manager: AdapterManager): Promise<SyncResult> {
  // 1. 并行从所有来源发现 Agent（网关可能离线，不报错）
  const [gatewayAgents, diskAgents, configAgents] = await Promise.all([
    discoverFromGateway(manager),
    Promise.resolve(discoverFromDisk()),
    Promise.resolve(discoverFromConfig()),
  ])

  // 2. 按优先级合并：config（最低） → disk → gateway（最高）
  const discovered = new Map<string, DiscoveredAgent>()
  for (const a of configAgents) discovered.set(a.id, a)
  for (const a of diskAgents) discovered.set(a.id, a)
  for (const a of gatewayAgents) discovered.set(a.id, a)

  // 3. 与数据库现有记录对比，计算差异
  const dbAgents = getAllAgents()
  const dbMap = new Map(dbAgents.map(a => [a.id, a]))

  // 4. 执行写入：新增 / 更新 / 标记离线
  for (const [id, agent] of discovered) {
    const existing = dbMap.get(id)
    if (!existing) {
      // 数据库中不存在 → 新增
      upsertAgent({ ...agent, lastSyncAt: now })
    } else {
      // 已存在 → 检查是否有变更
      const changes = diffAgent(existing, agent)
      if (changes.length > 0) {
        upsertAgent({ ...agent, id, lastSyncAt: now })
      }
    }
  }

  // 5. 处理已消失的 Agent：在数据库中存在但未被任何来源发现
  for (const [id, dbAgent] of dbMap) {
    if (!discovered.has(id) && dbAgent.status !== 'offline') {
      markAgentOffline(id)
    }
  }

  return result
}
```

### 条件分支说明

非显而易见的条件判断必须注释原因：

```ts
// 网关守卫：Agent 列表查询和同步操作允许在网关离线时访问
// 因为本地数据库有缓存数据，不依赖网关实时连接
if (c.req.path.startsWith('/api/agents') && c.req.method === 'GET') return next()
if (c.req.path.startsWith('/api/agents/sync')) return next()
```

### 错误处理说明

try-catch 块必须注释为什么忽略或如何处理错误：

```ts
try {
  gatewayAgent = await active.createAgent(params)
} catch {
  // 网关离线时忽略错误，仅创建本地记录
  // Agent 将在下次同步时推送到网关
}
```

## 数据库 Schema 注释

Drizzle schema 中每个表和关键字段都需要注释：

```ts
/** Agent 信息表 — 存储从所有来源同步的 Agent 数据 */
export const agents = pgTable('agents', {
  /** Agent 唯一标识，网关 Agent 使用网关分配的 ID，本地 Agent 使用 local-{目录名} */
  id: text('id').primaryKey(),
  /** Agent 显示名称 */
  name: text('name').notNull(),
  /** 当前状态：idle=空闲 / running=运行中 / error=异常 / offline=离线 */
  status: text('status').default('idle'),
  // ...
})
```

## 不需要注释的情况

- 变量名已经完全自解释：`const isLoading = true`
- 简单的 getter/setter
- 标准的 CRUD 路由（如果逻辑直白）
- import 语句

## 禁止的注释

- 无意义的重复注释：`// 设置名称` → `setName(value)`
- 注释掉的代码（直接删除）
- TODO 注释（要么现在做，要么记录到 issue）
