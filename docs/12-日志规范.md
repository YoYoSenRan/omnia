# 12 - 日志规范

## 日志库

使用 **pino**（高性能结构化日志库），开发环境配合 **pino-pretty** 美化输出。

```json
{
  "dependencies": {
    "pino": "^9.6.0"
  },
  "devDependencies": {
    "pino-pretty": "^13.0.0"
  }
}
```

## 日志实例

每个服务创建独立的 logger，带服务标识：

```ts
/**
 * 创建服务级日志实例
 *
 * 开发环境使用 pino-pretty 格式化输出，
 * 生产环境输出 JSON 格式（便于日志采集）。
 */
import pino from 'pino'

export const logger = pino({
  name: 'openclaw',  // 或 'server'
  level: process.env.LOG_LEVEL ?? 'info',
  transport: process.env.NODE_ENV === 'development'
    ? { target: 'pino-pretty', options: { colorize: true } }
    : undefined,
})
```

## 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| `fatal` | 服务无法继续运行 | 数据库连接失败、端口被占用 |
| `error` | 操作失败但服务仍可运行 | API 请求处理异常、同步写入失败 |
| `warn` | 潜在问题，不影响当前操作 | 网关断线、重试中、配置缺失使用默认值 |
| `info` | 关键业务事件 | 服务启动、网关连接成功、同步完成、任务状态变更 |
| `debug` | 调试信息 | 请求/响应详情、SQL 查询、事件触发 |
| `trace` | 极详细信息 | WebSocket 帧、SSE 推送内容 |

### 各环境默认级别

| 环境 | 默认级别 |
|------|---------|
| development | `debug` |
| production | `info` |
| test | `warn` |

## 结构化日志格式

所有日志必须使用结构化字段，禁止拼接字符串：

```ts
// ✅ 正确：结构化字段
logger.info({ agentId: 'agent-001', source: 'gateway' }, 'Agent synced')

// ❌ 错误：字符串拼接
logger.info(`Agent agent-001 synced from gateway`)
```

### 标准字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `service` | string | 服务名（pino name 自动填充） |
| `reqId` | string | 请求 ID（中间件自动生成） |
| `method` | string | HTTP 方法 |
| `path` | string | 请求路径 |
| `status` | number | HTTP 状态码 |
| `duration` | number | 请求耗时（毫秒） |
| `err` | object | 错误对象（pino 自动序列化） |

### 业务字段

| 字段 | 类型 | 用途 |
|------|------|------|
| `agentId` | string | Agent 相关操作 |
| `taskId` | string | Task 相关操作 |
| `sessionId` | string | Session 相关操作 |
| `source` | string | 数据来源（gateway / local / config） |
| `action` | string | 操作类型（sync / create / update / delete） |

## 请求日志中间件

每个 HTTP 请求自动记录入口和出口日志：

```ts
/**
 * 请求日志中间件
 *
 * 为每个请求分配唯一 reqId，记录请求开始和结束，
 * 包含方法、路径、状态码和耗时。
 */
import { randomUUID } from 'node:crypto'

app.use('*', async (c, next) => {
  const reqId = randomUUID().slice(0, 8)
  const start = Date.now()

  logger.info({
    reqId,
    method: c.req.method,
    path: c.req.path,
  }, 'Request started')

  await next()

  logger.info({
    reqId,
    method: c.req.method,
    path: c.req.path,
    status: c.res.status,
    duration: Date.now() - start,
  }, 'Request completed')
})
```

## 各场景日志示例

### 服务启动

```ts
logger.info({ port: 3001 }, 'Omnia openclaw service started')
logger.info({ port: 3002 }, 'Omnia server started')
```

### 网关连接

```ts
logger.info({ url: gatewayUrl }, 'Connecting to OpenClaw Gateway')
logger.info({ protocol: 3, tickMs: 30000 }, 'Gateway connected')
logger.warn({ url: gatewayUrl, retryIn: 5000 }, 'Gateway disconnected, will retry')
logger.error({ url: gatewayUrl, err }, 'Gateway connection failed')
```

### 数据同步

```ts
logger.info({ source: 'gateway', count: 5 }, 'Agent sync started')
logger.info({
  created: 2,
  updated: 1,
  removed: 0,
  unchanged: 2,
}, 'Agent sync completed')
logger.warn({ agentId: 'agent-001' }, 'Agent marked offline, not found in any source')
```

### 任务生命周期

```ts
logger.info({ taskId, title, assignTo }, 'Task created')
logger.info({ taskId, agentId, status: 'running' }, 'Task dispatched to agent')
logger.info({ taskId, status: 'done', duration: 45000 }, 'Task completed')
logger.error({ taskId, agentId, err }, 'Task execution failed')
```

### 错误日志

```ts
// pino 自动序列化 Error 对象，包含 stack trace
logger.error({ err, agentId }, 'Failed to sync agent')

// 业务错误（不含 stack）
logger.warn({ code: 10001, agentId }, 'Agent not found')
```

## 子日志

为不同模块创建子 logger，自动携带模块标识：

```ts
/** 网关模块日志 */
const gwLogger = logger.child({ module: 'gateway' })

/** 同步模块日志 */
const syncLogger = logger.child({ module: 'sync' })

/** 调度模块日志 */
const schedulerLogger = logger.child({ module: 'scheduler' })

// 使用时自动带上 module 字段
syncLogger.info({ agentId }, 'Agent synced')
// 输出: {"module":"sync","agentId":"agent-001","msg":"Agent synced"}
```
