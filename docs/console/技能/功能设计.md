# 技能 功能设计

路由：`/skills`

## 现状

- 表格式列表（名称+描述、来源、更新时间）
- 支持加载中/错误/空状态
- 无 Hooks 层（直接在页面内 fetch 或尚未实现）

---

## 一、列表页

### 1.1 搜索

列表顶部搜索框，按技能名称前端模糊过滤。

- placeholder："搜索技能..."
- 实时过滤（debounce 300ms）

### 1.2 创建技能

#### 入口

列表页右上角 "新建" 按钮，弹出 Dialog。

#### 表单字段

| 字段 | 组件 | 必填 | 校验规则 | 默认值 |
|------|------|------|----------|--------|
| name | Input | 是 | 非空，至少 1 字符 | — |
| description | Textarea | 否 | — | — |

#### 提交

1. Zod 校验
2. 调用 `POST /api/skills`（`useCreateSkill` mutation）
3. 成功：Toast "技能创建成功" → 关闭 Dialog → 刷新列表
4. 失败：Toast 错误信息

### 1.3 编辑技能

#### 入口

两种方式（任选其一）：
- **方案 A**：点击行项弹出编辑 Dialog（与创建复用表单，回填数据）
- **方案 B**：点击行项展开侧边 Drawer，展示详情 + 可编辑

建议 **方案 A**（Dialog），因为 Skill 字段少，Drawer 显得空旷。

#### 可编辑字段

| 字段 | 说明 |
|------|------|
| name | 技能名称 |
| description | 描述 |

#### 提交

调用 `PUT /api/skills/:id`（`useUpdateSkill` mutation）。

### 1.4 删除技能

#### 入口

每行末尾显示删除图标按钮。

#### 流程

1. 点击 → 确认 Dialog："确定删除技能「{name}」吗？"
2. 确认后调用 `DELETE /api/skills/:id`
3. 成功：Toast → 刷新列表

### 1.5 行项信息增强

当前行项展示较简洁，增加以下信息：

- `sourceRef`：来源引用（如有），灰色小字展示
- `contentHash`：内容哈希前 8 位（如有），用于判断是否有变更

## 二、Hooks 层

新建 `hooks/use-skills.ts`：

| Hook | 方法 | 路径 | 说明 |
|------|------|------|------|
| `useSkills()` | GET | `/api/skills` | 获取全部 |
| `useSkill(id)` | GET | `/api/skills/:id` | 获取单个（条件查询，id 存在时才发请求） |
| `useCreateSkill()` | POST | `/api/skills` | 创建，成功后 invalidate skills query |
| `useUpdateSkill(id)` | PUT | `/api/skills/:id` | 更新，成功后 invalidate skills + skill query |
| `useDeleteSkill()` | DELETE | `/api/skills/:id` | 删除，成功后 invalidate skills query |

参照 `use-agents.ts` 的实现模式。
