# 智能体 功能设计

路由：`/agents`（列表）、`/agents/:agentId`（详情）

## 现状

- 列表页：卡片式展示（emoji、名称、模型、状态徽章、来源徽章），支持加载中/错误/空状态
- 详情页：基本信息 11 字段只读展示 + Soul 只读 textarea + 审计日志时间线
- Hooks：`use-agents.ts` 已实现全部 CRUD hooks

---

## 一、列表页

### 1.1 搜索

列表顶部增加搜索框，按 Agent 名称前端模糊过滤。

- 输入框 placeholder："搜索智能体..."
- 实时过滤（onChange debounce 300ms），不调后端
- 搜索关键词高亮显示（可选）

### 1.2 状态筛选

搜索框右侧，标签组（Tab / SegmentedControl）切换：

```
[全部] [在线] [空闲] [忙碌] [错误] [离线]
```

- "全部" 不过滤
- "在线" = idle + busy
- 其余按单个状态过滤
- 筛选与搜索可叠加

### 1.3 来源筛选

状态筛选下方或右侧，下拉选择：

```
来源：[全部 ▾]  Gateway / Local / Config
```

### 1.4 创建智能体

#### 入口

列表页右上角 "新建" 按钮，点击弹出 Dialog。

#### 表单字段

| 字段 | 组件 | 必填 | 校验规则 | 默认值 |
|------|------|------|----------|--------|
| name | Input | 是 | 非空，至少 1 字符 | — |
| emoji | Input | 否 | — | — |
| role | Select | 否 | agent / coordinator / reviewer | agent |
| model | Input | 否 | — | — |
| workspace | Input | 否 | — | — |
| soul | Textarea | 否 | — | — |

#### 校验

使用 React Hook Form + Zod schema，与后端 `AgentCreateSchema` 对齐。

#### 提交流程

1. 前端 Zod 校验
2. 调用 `POST /api/agents`（`useCreateAgent` mutation）
3. 成功：Toast "智能体创建成功" → 关闭 Dialog → 自动刷新列表
4. 失败：Toast 错误信息，表单保持打开

### 1.5 删除智能体

#### 入口

每张卡片右上角更多菜单（⋮）→ "删除"，或 hover 时展示删除图标按钮。

#### 流程

1. 点击删除 → 打开确认 Dialog
2. 确认文案："确定删除智能体「{name}」吗？关联的审计日志不会被删除，但此操作不可恢复。"
3. 确认后调用 `DELETE /api/agents/:id`（`useDeleteAgent` mutation）
4. 成功：Toast → 刷新列表
5. 失败：Toast 错误信息

### 1.6 卡片点击

点击卡片跳转到详情页 `/agents/:agentId`（已实现）。

## 二、详情页

### 2.1 信息编辑

#### 入口

详情页标题栏右侧增加 "编辑" 按钮。

#### 交互方式

点击后弹出编辑 Dialog（与创建 Dialog 复用同一个表单组件，区别在于回填已有数据）。

#### 可编辑字段

| 字段 | 说明 |
|------|------|
| name | 名称 |
| emoji | 图标 |
| role | 角色 |
| model | 模型 |
| workspace | 工作目录 |

> `status`、`source` 等字段不在此编辑（status 由系统管理，source 由同步决定）。

#### 提交

调用 `PUT /api/agents/:id`（`useUpdateAgent` mutation）。成功后 Toast + 刷新详情数据。

### 2.2 Soul 编辑

#### 现状

只读 textarea 展示 Soul 内容。

#### 改造

- textarea 改为可编辑
- 底部增加 "保存" 按钮，初始状态禁用
- 内容变更后按钮变为可点击
- 点击保存调用 `PUT /api/agents/:id/soul`
- 成功后 Toast "Soul 已保存"
- 按钮恢复禁用状态

#### 需要新增的 Hook

在 `use-agents.ts` 中增加：

```
useUpdateSoul(id) — PUT /api/agents/:id/soul, body: { soul: string }
```

### 2.3 删除

详情页底部或标题栏增加 "删除" 按钮（destructive variant）。

流程同列表页删除，确认后调用 `DELETE /api/agents/:id`，成功后跳转到列表页 `/agents`。

### 2.4 审计日志

已实现，无需改动。可后续增加翻页加载（`limit` + `offset`）。

## 三、同步功能

### 入口

Agent 列表页顶部工具栏增加 "同步" 按钮（与 "新建" 并列）。

### 流程

```
用户点击 "同步"
    ↓
调用 POST /api/agents/sync/preview
    ↓
弹出预览 Dialog
    ┌──────────────────────────────┐
    │  同步预览                     │
    │                              │
    │  新增 (2)                     │
    │  • agent-a  [gateway]        │
    │  • agent-b  [local]          │
    │                              │
    │  更新 (1)                     │
    │  • agent-c  [config]         │
    │                              │
    │  下线 (0)                     │
    │                              │
    │       [取消]    [确认同步]    │
    └──────────────────────────────┘
    ↓ 确认
调用 POST /api/agents/sync
    ↓
Toast "同步完成：新增 2，更新 1"
刷新列表
```

### 空结果

预览结果全部为空时：Dialog 显示 "没有需要同步的变更"，只有关闭按钮。

### 需要新增的 Hooks

```
useSyncPreview()  — POST /api/agents/sync/preview
useSync()         — POST /api/agents/sync
```
