# 仪表盘 功能设计

路由：`/`

## 现状

静态占位页面，仅显示标题和副标题，无实际数据。

---

## 页面布局

采用经典上下结构，三层铺开：

```
┌─────────────────────────────────────────────────────────────┐
│  仪表盘                          [+ 新建智能体] [+ 新建任务] │
├─────────────┬─────────────┬─────────────┬───────────────────┤
│  ● 智能体   │  ⚡ 技能     │  ☐ 任务     │  💬 会话          │
│  3 在线     │  12         │  5 进行中    │  2 活跃           │
│  共 5 个    │  共 12 个   │  共 18 个    │  共 7 个          │
├─────────────┴─────────────┴─────────────┴───────────────────┤
│ ┌─ 系统状态 ────────────────────────────────────────────┐   │
│ │  Backend: ● 正常    Gateway: ● 已连接    ⟳ 触发同步   │   │
│ └───────────────────────────────────────────────────────┘   │
├──────────────────────────────────┬──────────────────────────┤
│                                  │                          │
│  最近活动                         │  Agent 状态一览          │
│  ● agent.created — 3 分钟前       │  🤖 Coder      ● idle   │
│    创建了智能体 "Coder"            │  🔍 Reviewer   ● busy   │
│  ● task.updated — 10 分钟前       │  ✍️ Writer     ● offline │
│    任务状态变更为 done             │                          │
│  ● session.created — 30 分钟前    │──────────────────────────│
│    ...                           │  任务分布                 │
│                                  │  done     ████████░ 62%  │
│                                  │  wip      ███░░░░░░ 23%  │
│                                  │  inbox    █░░░░░░░░  8%  │
│                                  │  failed   ░░░░░░░░░  7%  │
└──────────────────────────────────┴──────────────────────────┘
```

### 布局规格

| 区域 | 宽度 | 高度 | 说明 |
|------|------|------|------|
| 统计卡片行 | 等宽 4 列 `grid-cols-4` | 自适应 | 响应式：`<768px` 折为 `grid-cols-2` |
| 系统状态条 | 全宽 | 固定 | 横向排列，紧凑样式 |
| 最近活动 | 约 60% `col-span-7` | 自适应 | 左侧主区域 |
| 右侧面板 | 约 40% `col-span-5` | 自适应 | Agent 状态 + 任务分布上下叠放 |

容器网格：`grid grid-cols-12 gap-6`

## 功能点

### 1. 统计卡片

页面顶部一行 4 张卡片，实时展示系统概况。

| 卡片 | 图标 | 主数值 | 副数值 | 数据来源 |
|------|------|--------|--------|----------|
| 智能体 | `Bot` | 在线数 | 总数 | `GET /api/agents`，`status !== 'offline'` 为在线 |
| 技能 | `Zap` | 总数 | — | `GET /api/skills`，取数组长度 |
| 任务 | `ListTodo` | 进行中 | 总数 | `GET /api/tasks`，`status in (assigned, in_progress, review)` |
| 会话 | `MessageSquare` | 活跃数 | 总数 | `GET /api/sessions`，`status === 'open'` |

#### 卡片组件结构

```
┌────────────────────────┐
│  [图标]  智能体          │
│                        │
│  3 在线                 │  ← text-2xl font-semibold text-foreground
│  共 5 个               │  ← text-sm text-muted-foreground
└────────────────────────┘
```

- 容器：`bg-card border border-border rounded-lg p-4`
- 图标 + 标题同行，图标 `size-4 text-muted-foreground`，标题 `text-sm font-medium text-muted-foreground`
- 主数值：`text-2xl font-semibold tracking-tight text-foreground`
- 副数值：`text-xs text-muted-foreground`

#### 数据获取

复用各模块 hooks（`useAgents` / `useSkills` / `useTasks` / `useSessions`），在 Dashboard 中聚合计算。不额外调接口。

#### 刷新策略

依赖 TanStack Query 缓存 + SSE 事件自动失效，无需手动刷新。

---

### 2. 系统状态条

统计卡片下方，紧凑的状态指示条，展示后端服务和 Gateway 连接状态。

#### 数据来源

| 指标 | 接口 | 正常条件 |
|------|------|----------|
| Backend | `GET /health` | HTTP 200 |
| Gateway | `GET /api/gateway/status` | `status === 'connected'` |

#### 展示

```
┌─────────────────────────────────────────────────────────┐
│  Backend: ● 正常    Gateway: ● 已连接    [⟳ 触发同步]   │
└─────────────────────────────────────────────────────────┘
```

- 容器：`bg-card border border-border rounded-lg px-4 py-2`
- 状态点：正常 `text-emerald-500`，异常 `text-destructive`，连接中 `text-yellow-500 animate-pulse`
- "触发同步" 按钮放在右侧，`variant="outline" size="sm"`

#### 轮询策略

`refetchInterval: 30_000`（30 秒），后台页面暂停轮询（`refetchIntervalInBackground: false`）。

---

### 3. 最近活动

左侧主区域，展示系统最近操作流。

#### 数据来源

`GET /api/activities?limit=20`

#### 展示形式

竖向时间线，每条记录一行：

```
● agent.created — user — 3 分钟前
  创建了智能体 "Coder"

● task.updated — system — 10 分钟前
  任务状态变更为 done

● agent.synced — sync — 1 小时前
  同步完成：新增 2，更新 1
```

每条活动：
- 左侧竖线 + 圆点（`border-l-2 border-border`）
- 第一行：动作标签（`text-xs font-mono bg-muted rounded px-1`）+ 来源 + 相对时间
- 第二行：detail 解析后的可读描述，`text-sm text-muted-foreground`

#### 交互

- 实体名称可点击，跳转到对应详情页（Agent → `/agents/:id`，Task → 打开详情抽屉）
- hover 时间显示绝对时间 tooltip
- detail 字段为 JSON，解析后展示关键信息（如 `{ name: "Coder" }` → `创建了智能体 "Coder"`）

#### 空状态

使用 `EmptyState` 组件，图标 `Activity`，标题 "暂无最近活动"。

---

### 4. Agent 状态一览

右侧面板上半部分，展示所有 Agent 及其实时状态。

#### 数据来源

复用 `useAgents()` 返回的列表。

#### 展示

每个 Agent 一行：

```
🤖 Coder       ● idle
🔍 Reviewer    ● busy
✍️ Writer      ● offline
```

- Agent emoji（或 `Bot` 图标 fallback） + 名称 + 右侧状态点
- 状态颜色：`idle` 灰 / `busy` 绿 / `error` 红 / `offline` 黄
- 整行可点击，跳转 `/agents/:id`
- hover：`hover:bg-accent rounded-md transition-colors`

#### Agent 数量多时

超过 8 个显示前 8 个 + "查看全部 →" 链接跳转 `/agents`。

---

### 5. 任务分布

右侧面板下半部分，按状态展示任务分布。

#### 数据来源

复用 `useTasks()` 返回的列表，前端 `reduce` 按状态分组计数。

#### 展示

水平条形图，纯 Tailwind 实现（不引入 chart 库）：

```
任务分布

done       ████████████████░░░░  62%  (8)
in_progress ██████░░░░░░░░░░░░░░  23%  (3)
inbox      ██░░░░░░░░░░░░░░░░░░   8%  (1)
failed     █░░░░░░░░░░░░░░░░░░░   7%  (1)
```

- 每行：状态标签（StatusBadge 复用）+ 进度条 + 百分比 + 数量
- 进度条：`bg-muted rounded-full h-2`，填充色按状态取语义色
- 状态色映射：`done` 绿 / `in_progress` 蓝 / `inbox` 灰 / `assigned` 蓝 / `review` 紫 / `failed` 红
- 仅展示数量 > 0 的状态

#### 空状态

无任务时显示 "暂无任务" + [新建任务] 按钮。

---

### 6. 快捷操作

标题栏右侧放置快捷操作按钮：

| 操作 | 按钮样式 | 触发 | 说明 |
|------|---------|------|------|
| 新建智能体 | `variant="default" size="sm"` | 打开 Agent 创建 Dialog | 复用智能体模块的创建弹窗 |
| 新建任务 | `variant="outline" size="sm"` | 打开 Task 创建 Dialog | 复用任务模块的创建弹窗 |
| 触发同步 | 系统状态条内 | 调用 `POST /api/agents/sync` | 结果 Toast 提示 |
