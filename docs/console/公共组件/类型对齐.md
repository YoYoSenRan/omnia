# 前后端类型对齐

## 概述

前端 `types/index.ts` 中的类型定义需要与后端 `db/schema/` 保持一致。以下是当前的差异和修复方案。

---

## 一、TaskStatus 状态值不一致

**问题**：前端和后端使用了不同的状态命名。

| 前端（当前） | 后端（实际） | 说明 |
|-------------|-------------|------|
| `pending` | `inbox` | 命名不同 |
| `assigned` | `assigned` | ✅ 一致 |
| `running` | `in_progress` | 命名不同 |
| — | `review` | 前端缺失 |
| `completed` | `done` | 命名不同 |
| `failed` | `failed` | ✅ 一致 |
| `cancelled` | `cancelled` | ✅ 一致 |

**修复**：

```ts
// types/index.ts
export type TaskStatus = "inbox" | "assigned" | "in_progress" | "review" | "done" | "failed" | "cancelled"
```

同时更新 `lib/constants.ts` 中的 `TASK_STATUS_MAP`：

```ts
export const TASK_STATUS_MAP = {
  inbox: { label: "收件箱", color: "text-gray-500" },
  assigned: { label: "已分配", color: "text-blue-600" },
  in_progress: { label: "进行中", color: "text-green-600" },
  review: { label: "待审查", color: "text-purple-600" },
  done: { label: "已完成", color: "text-emerald-600" },
  failed: { label: "失败", color: "text-red-600" },
  cancelled: { label: "已取消", color: "text-gray-400" },
}
```

看板页 `views/tasks/index.tsx` 的分组逻辑也需要对应更新。

---

## 二、Agent 类型缺少 role 字段

**问题**：后端 `agents` 表有 `role` 字段（agent / coordinator / reviewer），前端 `Agent` 接口缺失。

**修复**：

```ts
// types/index.ts
export interface Agent {
  id: string
  name: string
  emoji: string | null
  role: string | null         // 新增
  model: string | null
  workspace: string | null
  status: AgentStatus
  source: AgentSource
  sourceRef: string | null
  soul: string | null
  config: unknown             // 改为 unknown（后端是 jsonb）
  contentHash: string | null  // 新增
  lastSyncAt: string | null
  lastActiveAt: string | null
  createdAt: string
  updatedAt: string
}
```

---

## 三、Task 类型缺少 priority 和 tags 字段

**问题**：后端 `tasks` 表有 `priority`（整数 1-5）和 `tags`（字符串数组），前端 `Task` 接口缺失。

**修复**：

```ts
// types/index.ts
export interface Task {
  id: string
  title: string
  description: string | null
  status: TaskStatus
  priority: number             // 新增
  assignedTo: string | null
  result: string | null
  parentId: string | null
  tags: string[] | null        // 新增
  createdAt: string
  updatedAt: string
  completedAt: string | null
}
```

---

## 四、Session 类型缺少 metadata 字段

**问题**：后端 `sessions` 表有 `metadata`（jsonb），前端 `Session` 接口缺失。

**修复**：

```ts
// types/index.ts
export interface Session {
  id: string
  agentId: string
  status: SessionStatus
  metadata: unknown            // 新增
  createdAt: string
  closedAt: string | null
}
```

---

## 五、AgentStatus 值不一致

**问题**：前端使用 `running`，后端使用 `busy`。

| 前端（当前） | 后端（实际） |
|-------------|-------------|
| `idle` | `idle` |
| `running` | `busy` |
| `error` | `error` |
| `offline` | `offline` |

**修复**：

```ts
// types/index.ts
export type AgentStatus = "idle" | "busy" | "error" | "offline"
```

同时更新 `lib/constants.ts` 中的 `AGENT_STATUS_MAP`：

```ts
export const AGENT_STATUS_MAP = {
  idle: { label: "空闲", color: "text-gray-500" },
  busy: { label: "忙碌", color: "text-green-600" },
  error: { label: "错误", color: "text-red-600" },
  offline: { label: "离线", color: "text-yellow-600" },
}
```
