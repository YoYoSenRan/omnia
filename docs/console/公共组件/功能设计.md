# 公共组件 功能设计

## 概述

各模块共用的 UI 组件和工具函数。目标是消除页面间的重复实现，统一交互体验。

---

## 一、Toast 通知

### 方案

使用已安装的 Sonner。

### 集成

1. 在 `App.tsx` 根组件中挂载 `<Toaster />`
2. 配置位置：右上角（`position="top-right"`）
3. 深色模式跟随 `resolvedTheme`

### 使用规范

| 场景 | 调用 |
|------|------|
| 操作成功 | `toast.success("智能体创建成功")` |
| 操作失败 | `toast.error("操作失败：" + error.message)` |
| 提示信息 | `toast.info("同步进行中...")` |

所有 TanStack Query mutation 的 `onSuccess` / `onError` 回调中统一调用。

## 二、确认弹窗 ConfirmDialog

### 用途

删除、关闭等不可逆操作的二次确认。

### Props

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| open | boolean | 是 | 是否打开 |
| onOpenChange | (open: boolean) => void | 是 | 开关回调 |
| title | string | 是 | 标题，如 "确认删除" |
| description | string | 是 | 说明文案 |
| confirmText | string | 否 | 确认按钮文字，默认 "确认" |
| cancelText | string | 否 | 取消按钮文字，默认 "取消" |
| variant | "default" \| "destructive" | 否 | 确认按钮样式，默认 "destructive" |
| loading | boolean | 否 | 确认按钮加载状态 |
| onConfirm | () => void | 是 | 确认回调 |

### 结构

```
┌──────────────────────────┐
│  确认删除                 │
│                          │
│  确定删除智能体「Coder」  │
│  吗？此操作不可恢复。     │
│                          │
│       [取消]   [删除]    │
└──────────────────────────┘
```

### 实现

基于 Base UI Dialog 或自建，遮罩 + 居中弹窗。

## 三、Dialog 弹窗

### 用途

创建/编辑表单的容器。

### Props

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| open | boolean | 是 | 是否打开 |
| onOpenChange | (open: boolean) => void | 是 | 开关回调 |
| title | string | 是 | 标题 |
| children | ReactNode | 是 | 内容区 |

### 行为

- ESC 键关闭
- 点击遮罩关闭
- 内容区可滚动（表单字段多时）
- 关闭时不清空表单状态（由调用方控制）

## 四、Drawer 侧边抽屉

### 用途

详情展示、轻量编辑（Task 详情、Session 详情、Skill 编辑）。

### Props

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| open | boolean | 是 | 是否打开 |
| onOpenChange | (open: boolean) => void | 是 | 开关回调 |
| title | string | 否 | 标题 |
| children | ReactNode | 是 | 内容区 |
| footer | ReactNode | 否 | 底部操作栏 |
| width | string | 否 | 宽度，默认 "480px" |

### 行为

- 从右侧滑入，带 transition 动画
- 左侧遮罩，点击关闭
- ESC 键关闭
- 内容区独立滚动
- footer 固定底部

### 结构

```
                    ┌──────────────┐
                    │ × 标题       │
                    │              │
                    │  内容区      │
                    │  （可滚动）  │
                    │              │
                    │──────────────│
                    │ [操作按钮]   │
                    └──────────────┘
```

## 五、表单组件

配合 React Hook Form 使用，统一 label + input + error 布局。

### FormField 容器

```tsx
<FormField label="名称" error={errors.name?.message} required>
  <Input {...register("name")} />
</FormField>
```

渲染为：
```
名称 *
┌──────────────────────┐
│                      │
└──────────────────────┘
名称不能为空
```

### 组件清单

| 组件 | 说明 | 使用场景 |
|------|------|----------|
| `FormField` | label + error 容器 | 所有表单字段 |
| `Input` | 文本输入框 | name、model、workspace 等 |
| `Textarea` | 多行文本框 | description、soul |
| `Select` | 下拉选择 | role、priority、assignedTo |
| `TagInput` | 标签输入 | tasks.tags |

### Input

基础文本输入框，支持：
- placeholder
- disabled 状态
- 错误态边框（红色）
- 与 React Hook Form `register` 兼容

### Textarea

多行文本框，支持：
- 自动增高或固定高度
- 字符计数（可选）
- 其余同 Input

### Select

下拉选择，支持：
- options: `{ value: string, label: string }[]`
- placeholder
- disabled 状态
- 与 React Hook Form `Controller` 兼容

### TagInput

标签输入组件，支持：
- 输入文字 + Enter 添加标签
- 标签旁 × 按钮删除
- 已有标签以 pill 形式展示
- 与 React Hook Form `Controller` 兼容

## 六、状态徽章 StatusBadge

### 现状

各页面内部各自定义了 StatusBadge / SourceBadge，样式和逻辑重复。

### 统一组件

```tsx
<StatusBadge status="idle" type="agent" />
<StatusBadge status="in_progress" type="task" />
<StatusBadge status="open" type="session" />
```

### Props

| 属性 | 类型 | 说明 |
|------|------|------|
| status | string | 状态值 |
| type | "agent" \| "task" \| "session" | 实体类型，决定颜色映射 |

### 颜色映射

从 `lib/constants.ts` 中的 `XXX_STATUS_MAP` 读取。

## 七、空状态 EmptyState

### 现状

各页面各写各的空状态（文字+样式不统一）。

### 统一组件

```tsx
<EmptyState
  icon={<InboxIcon />}
  title="暂无智能体"
  description="点击右上角按钮创建你的第一个智能体"
  action={<Button onClick={openCreate}>新建智能体</Button>}
/>
```

### Props

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| icon | ReactNode | 否 | 顶部图标 |
| title | string | 是 | 主标题 |
| description | string | 否 | 描述文案 |
| action | ReactNode | 否 | 操作按钮 |

### 样式

居中布局，灰色调，图标 48px，标题 text-lg，描述 text-sm text-muted。

## 八、时间格式化

### 需求

所有时间字段统一为相对时间展示，hover 显示绝对时间。

### 实现

封装 `RelativeTime` 组件或 `useRelativeTime` hook：

```tsx
<RelativeTime date="2024-01-15T10:30:00Z" />
// 渲染为：3 分钟前（hover tooltip：2024-01-15 10:30:00）
```

### 规则

| 时间差 | 展示 |
|--------|------|
| < 1 分钟 | 刚刚 |
| < 60 分钟 | N 分钟前 |
| < 24 小时 | N 小时前 |
| < 30 天 | N 天前 |
| >= 30 天 | yyyy-MM-dd |
