# 会话 功能设计

路由：`/sessions`

## 现状

- 表格式列表（ID 前 8 位、Agent ID 前 8 位、状态、创建时间、关闭时间）
- 支持加载中/错误/空状态
- 无 Hooks 层

---

## 一、列表页

### 1.1 筛选

列表顶部增加状态筛选标签：

```
[全部] [进行中] [已关闭]
```

- "全部" 不过滤
- "进行中" = status === 'open'
- "已关闭" = status === 'closed'

### 1.2 列表增强

#### 表头

| 列 | 宽度 | 说明 |
|----|------|------|
| 会话 ID | 固定 | 前 8 位，hover tooltip 显示完整 ID，点击复制 |
| 关联智能体 | 自适应 | 展示 Agent 名称而非 ID（需关联查询） |
| 状态 | 固定 | StatusBadge 组件 |
| 创建时间 | 固定 | 相对时间，hover 显示绝对时间 |
| 关闭时间 | 固定 | 相对时间或 "—" |
| 操作 | 固定 | 关闭按钮（仅 open 状态） |

#### 关联 Agent 名称

方案：页面加载时同时调用 `useAgents()` 获取 Agent 列表，建立 `id → name` 映射表。在渲染每行时查表获取 Agent 名称。

如果 Agent 不存在（已删除），显示 ID 前 8 位 + "(已删除)" 灰色文字。

### 1.3 关闭会话

#### 入口

操作列中，status === 'open' 的行显示 "关闭" 按钮。已关闭的行该列为空。

#### 流程

1. 点击 "关闭" → 确认 Dialog："确定关闭此会话吗？关闭后不可重新打开。"
2. 调用 `PUT /api/sessions/:id/close`（`useCloseSession` mutation）
3. 成功：Toast "会话已关闭" → 刷新列表
4. 失败：Toast 错误信息（如 409 表示已关闭）

### 1.4 会话详情

#### 入口

点击行项（除操作列外）展开详情。

#### 方案选择

- **方案 A**：侧边 Drawer — 适合需要查看详细信息同时保持列表可见
- **方案 B**：详情页 `/sessions/:id` — 适合后续有消息记录等丰富内容

当前建议 **方案 A**（Drawer），Session 信息较少，Drawer 足够。

#### Drawer 内容

```
┌──────────────────────────────────┐
│  × 关闭                          │
│                                  │
│  会话详情                         │
│  ────────────────                │
│                                  │
│  ID        sess-abc12345         │
│  状态      [open]                │
│  关联智能体 Coder (→ 跳转链接)    │
│                                  │
│  ── Metadata ─────────────       │
│  { "key": "value" }             │
│  （格式化 JSON 展示）             │
│                                  │
│  ── 时间 ─────────────────       │
│  创建时间  2024-01-15 10:30      │
│  关闭时间  —                     │
│                                  │
│  ────────────────                │
│  [关闭会话]    (仅 open 时显示)  │
└──────────────────────────────────┘
```

- Agent 名称可点击跳转到 `/agents/:agentId`
- metadata 使用 `<pre>` 格式化展示 JSON
- 底部关闭按钮复用关闭逻辑

## 二、Hooks 层

新建 `hooks/use-sessions.ts`：

| Hook | 方法 | 路径 | 说明 |
|------|------|------|------|
| `useSessions()` | GET | `/api/sessions` | 获取全部 |
| `useSession(id)` | GET | `/api/sessions/:id` | 获取单个 |
| `useCloseSession()` | PUT | `/api/sessions/:id/close` | 关闭会话 |

> Session 不支持前端创建（由系统/Agent 自动创建），也不支持删除。所以不需要 useCreateSession / useDeleteSession。
