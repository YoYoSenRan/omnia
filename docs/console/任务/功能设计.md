# 任务 功能设计

路由：`/tasks`

## 现状

- 三列看板（待处理/进行中/已完成）
- 任务卡片（标题、状态标签、分配 Agent、创建时间）
- 支持加载中/错误/空状态
- 无 Hooks 层

---

## 一、看板

### 1.1 列定义

保持 3 列简洁布局，内部按后端状态分组：

| 列 | 包含的后端状态 | 列标题 |
|----|---------------|--------|
| 待处理 | inbox, assigned | 待处理 |
| 进行中 | in_progress, review | 进行中 |
| 已完成 | done, failed, cancelled | 已完成 |

> 后端状态已从 pending/running/completed 改为 inbox/in_progress/done 等，前端需对齐。

每列标题旁显示该列任务数量徽章。

### 1.2 卡片信息

每张任务卡片展示：

```
┌──────────────────────────┐
│ [优先级色块] 任务标题     │
│                          │
│ 📌 in_progress           │
│ 🤖 agent-coder           │
│ 🏷️ tag1, tag2            │
│                          │
│ 3 分钟前                  │
└──────────────────────────┘
```

- 左侧边框颜色表示优先级（P1 红 / P2 橙 / P3 蓝 / P4 灰 / P5 浅灰）
- 状态标签：使用统一 StatusBadge 组件
- 分配 Agent：显示 Agent 名称（需关联查询或展示 ID 前 8 位）
- 标签：tags 数组，逗号分隔
- 底部：相对时间

### 1.3 卡片点击

点击卡片打开详情抽屉（Drawer），见下文。

## 二、创建任务

### 入口

看板顶部右侧 "新建任务" 按钮，弹出 Dialog。

### 表单字段

| 字段 | 组件 | 必填 | 校验规则 | 默认值 |
|------|------|------|----------|--------|
| title | Input | 是 | 非空，至少 1 字符 | — |
| description | Textarea | 否 | — | — |
| priority | Select | 否 | 1-5 整数 | 3 |
| assignedTo | Select | 否 | 从 Agent 列表加载选项 | — |
| tags | TagInput | 否 | 字符串数组 | — |

#### assignedTo 下拉

选项来自 `useAgents()` 查询结果，展示 Agent emoji + 名称，值为 Agent ID。
第一项为 "不分配"（值为 null）。

#### tags 输入

输入关键词后按 Enter 添加标签，标签旁有 × 按钮可删除。

### 提交

1. Zod 校验
2. 调用 `POST /api/tasks`（`useCreateTask` mutation）
3. 成功：Toast "任务创建成功" → 关闭 Dialog → 刷新看板
4. 任务如果指定了 assignedTo，后端自动设状态为 assigned

## 三、任务详情抽屉

### 触发

点击看板卡片 → 右侧滑出 Drawer。

### 布局

```
┌─────────────────────────────────────────┐
│  × 关闭                                │
│                                         │
│  任务标题                                │
│  ──────────────────────────             │
│                                         │
│  状态    [inbox]  → [assigned] [cancelled]│
│  优先级  ★★★☆☆ (3)                      │
│  分配给  agent-coder                     │
│  标签    tag1, tag2                      │
│                                         │
│  ── 描述 ──────────────────────          │
│  这是任务的详细描述...                    │
│                                         │
│  ── 执行结果 ──────────────────          │
│  （仅 done/failed 状态时展示）            │
│  任务执行的返回结果...                    │
│                                         │
│  ── 时间信息 ──────────────────          │
│  创建时间  2024-01-15 10:30              │
│  更新时间  2024-01-15 11:00              │
│  完成时间  2024-01-15 11:30              │
│                                         │
│  ─────────────────────────────          │
│  [编辑]              [删除]              │
└─────────────────────────────────────────┘
```

### 状态流转按钮

根据当前状态动态展示可流转的目标状态，以按钮组形式排列：

| 当前状态 | 可流转到 |
|---------|---------|
| inbox | assigned, cancelled |
| assigned | in_progress, cancelled |
| in_progress | review, done, failed, cancelled |
| review | done, in_progress, failed |
| done | （终态，不可流转） |
| failed | inbox |
| cancelled | inbox |

每个按钮点击后调用 `PUT /api/tasks/:id` 更新状态。

- "done" / "cancelled" 按钮使用二次确认
- 状态更新成功后 Toast + 刷新看板

## 四、编辑任务

### 入口

详情抽屉底部 "编辑" 按钮，弹出 Dialog（复用创建表单，回填数据）。

### 可编辑字段

| 字段 | 说明 |
|------|------|
| title | 标题 |
| description | 描述 |
| priority | 优先级 |
| assignedTo | 分配 Agent |
| tags | 标签 |

> 状态不在编辑表单中修改，而是通过状态流转按钮。

### 提交

调用 `PUT /api/tasks/:id`（`useUpdateTask` mutation）。

---

## 五、删除任务

详情抽屉底部 "删除" 按钮（destructive variant）。

1. 确认 Dialog："确定删除任务「{title}」吗？"
2. 调用 `DELETE /api/tasks/:id`
3. 成功：Toast → 关闭抽屉 → 刷新看板

---

## 六、Hooks 层

新建 `hooks/use-tasks.ts`：

| Hook | 方法 | 路径 | 说明 |
|------|------|------|------|
| `useTasks()` | GET | `/api/tasks` | 获取全部 |
| `useTask(id)` | GET | `/api/tasks/:id` | 获取单个 |
| `useCreateTask()` | POST | `/api/tasks` | 创建 |
| `useUpdateTask(id)` | PUT | `/api/tasks/:id` | 更新（含状态流转） |
| `useDeleteTask()` | DELETE | `/api/tasks/:id` | 删除 |
