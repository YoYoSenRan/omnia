# 02 - 技术栈

## 前端（console + web）

| 技术 | 版本 | 用途 |
|------|------|------|
| React | 19 | UI 框架 |
| TypeScript | 5.7+ | 类型安全 |
| Vite | 6+ | 构建工具 |
| Tailwind CSS | 4+ | 样式 |
| shadcn/ui | latest | UI 组件库（各前端独立安装） |
| Zustand | 5+ | 状态管理 |
| TanStack Query | 5+ | 数据请求与缓存 |
| React Router | 7+ | 路由 |
| Framer Motion | 11+ | 动画（可选） |

### 前端约定

- console 和 web 各自独立，不共享组件库
- 各自独立安装 shadcn/ui 组件
- 各自维护自己的类型定义

## 后端（openclaw + server）

| 技术 | 版本 | 用途 |
|------|------|------|
| Node.js | 22+ | 运行时 |
| TypeScript | 5.7+ | 类型安全 |
| Hono | 4+ | HTTP 框架 |
| PostgreSQL | 16+ | 数据库 |
| Drizzle ORM | 0.38+ | ORM |
| drizzle-kit | 0.30+ | 数据库迁移工具 |
| tsx | 4+ | 开发时 TypeScript 直接运行 |

### 后端约定

- openclaw 和 server 各自有独立的 PostgreSQL 数据库
- 各自维护独立的 Drizzle schema 和迁移
- server 通过 HTTP 调用 openclaw 的 API，不直连 core 数据库

## 实时通信

| 技术 | 方向 | 用途 |
|------|------|------|
| WebSocket | openclaw ↔ OpenClaw Gateway | 网关协议通信 |
| SSE | openclaw → console / server | 事件实时推送 |
| SSE | server → web | 业务事件推送（可选） |
