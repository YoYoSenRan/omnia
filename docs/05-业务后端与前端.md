# 05 - 业务后端与前端（apps/server + apps/web）

## 业务后端（apps/server）

### 职责

垂直业务的后端服务，独立进程，端口 3002。通过 HTTP 调用 openclaw 服务获取 agent 能力。

### 目录结构

```
apps/server/src/
├── core-client/       # OpenClaw Core 客户端
│   ├── client.ts      #   封装对 openclaw 服务的 HTTP 调用
│   ├── events.ts      #   订阅 openclaw 的 SSE 事件
│   └── types.ts       #   响应类型定义
├── db/                # 数据库
│   ├── schema.ts      #   Drizzle schema（业务表）
│   ├── migrate.ts     #   迁移管理
│   └── index.ts       #   数据库连接
├── routes/            # HTTP 路由
│   └── ...            #   按业务模块组织
├── services/          # 业务逻辑
│   └── ...
├── lib/               # 工具函数
│   └── response.ts    #   统一响应格式
├── app.ts             # Hono 应用
└── index.ts           # 入口
```

### 数据库（omnia_app）

业务专属数据库，表结构根据垂直项目需求定义。与 openclaw 的数据库完全隔离。

### Core 客户端示例

```ts
// server 中调用 openclaw 服务
import { CoreClient } from './core-client/client'

const core = new CoreClient(process.env.OPENCLAW_URL)

// 获取可用 agents
const agents = await core.agents.list()

// 提交任务
const task = await core.tasks.create({
  title: '调研 AI IDE 竞品',
  assignTo: 'researcher',
})

// 查询任务结果
const result = await core.tasks.get(task.id)
```

## 业务前端（apps/web）

### 职责

垂直项目的用户界面，展示任务流程、agent 运行过程、产出内容。

### 目录结构

```
apps/web/src/
├── api/              # API 客户端
│   └── client.ts     #   封装对 server 的请求
├── components/       # 组件
│   └── ui/           #   shadcn/ui 组件（独立安装）
├── hooks/
├── pages/            # 页面（根据业务需求定义）
├── stores/           # Zustand 状态
├── layout/           # 布局
├── router.tsx
├── App.tsx
└── main.tsx
```

### 核心功能（首个垂直项目）

- 需求输入（对话式或表单）
- 任务拆解展示（树状结构）
- 执行进度追踪（实时）
- Agent 运行过程可视化
- 产出物展示与下载

### 数据流

```
用户输入需求
  → web 发送到 server
  → server 调 openclaw 拆解任务
  → server 调 openclaw 分配给 agents
  → openclaw 通过 SSE 推送进度
  → server 转发给 web
  → web 实时展示
```
