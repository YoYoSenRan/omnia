# 01 - 架构总览

## 系统定位

Omnia 是一个基于 OpenClaw 的智能体编排平台，采用 Core + 业务的分层架构：

- **Core 层**（openclaw + console）：稳定的基座，负责与 OpenClaw 网关对接，提供 agent 管理、任务调度、事件推送等通用能力。
- **业务层**（server + web）：垂直项目的具体实现，通过 Core 的 API 获取 agent 能力，构建具体的应用场景。

## 四个应用

### openclaw（Core 后端服务）

OpenClaw 对接服务，独立进程，端口 3001。

职责：
- OpenClaw 网关 WebSocket 连接管理
- Agent / Skill / Session 状态同步
- 任务调度引擎
- 事件总线 + SSE 推送
- PostgreSQL 持久化（agents、skills、sessions、tasks、logs）

对外暴露两套 API：
- **内部 API**：供 console 使用，全功能（CRUD、配置、系统管理）
- **开放 API**：供 server 使用，侧重查询和调度

### console（Core 前端）

控制台前端，与 openclaw 配套。

功能：
- 网关连接状态监控
- Agent 列表、详情、Soul 编辑
- Skill 浏览与管理
- 会话监控
- 任务状态看板
- 审计日志

### server（业务后端服务）

垂直项目的后端，独立进程，端口 3002。

职责：
- 垂直业务 API
- 通过 HTTP 调用 openclaw 获取 agent 能力
- 任务提交与结果处理
- 业务专属数据库（独立 PG 数据库）

### web（业务前端）

垂直项目的前端。

功能：
- 需求输入界面
- 任务拆解与执行流程展示
- Agent 运行过程可视化
- 产出物展示与管理

## 数据隔离

- openclaw 使用独立的 PostgreSQL 数据库（`omnia_core`）
- server 使用独立的 PostgreSQL 数据库（`omnia_app`）
- 两者数据完全隔离，server 通过 openclaw 的 API 获取 agent 数据，不直连 core 数据库

## 未来演进

1. **当前阶段**：monorepo，四个 app 在同一仓库开发
2. **成熟阶段**：openclaw + console 抽为 npm 包（`@omnia/core`、`@omnia/console`），业务项目独立仓库通过依赖引入
3. **多项目复用**：不同的垂直项目（AI IDE、需求雷达等）各自独立仓库，共享同一套 Core
