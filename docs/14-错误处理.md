# 14 - 错误处理

## 设计原则

- **快速失败**：在最早的位置暴露问题，不添加掩盖错误的 fallback
- **错误分类**：区分可恢复错误和致命错误，分别处理
- **上下文传递**：错误信息必须包含足够的排查上下文
- **统一出口**：所有错误通过统一的响应格式返回

## 错误分类

| 类型 | 处理方式 | 示例 |
|------|---------|------|
| 参数校验错误 | 返回 400，明确指出哪个字段有问题 | 缺少 name、格式非法 |
| 业务逻辑错误 | 返回对应状态码 + 业务 code | Agent 不存在、状态不允许 |
| 依赖服务错误 | 返回 503，记录 error 日志 | 网关离线、数据库连接失败 |
| 未捕获异常 | 返回 500，记录 error 日志 + 堆栈 | 代码 bug |

## 后端全局错误处理

```ts
/**
 * 全局错误处理器
 *
 * 捕获所有未处理的异常，统一转换为标准错误响应。
 * 生产环境不暴露错误堆栈。
 */
app.onError((err, c) => {
  // 已知业务错误（自定义 AppError）
  if (err instanceof AppError) {
    logger.warn({ code: err.code, err }, err.message)
    return fail(c, err.httpStatus, err.code, err.message)
  }

  // 未知错误
  logger.error({ err }, 'Unhandled error')
  return fail(
    c,
    500,
    CODE.INTERNAL_ERROR,
    process.env.NODE_ENV === 'development'
      ? err.message
      : 'Internal server error',
  )
})
```

### 自定义错误类

```ts
/**
 * 应用级业务错误
 *
 * 携带 HTTP 状态码和业务状态码，
 * 被全局错误处理器捕获后转换为标准响应。
 */
export class AppError extends Error {
  constructor(
    /** HTTP 状态码 */
    public httpStatus: number,
    /** 业务状态码 */
    public code: number,
    /** 错误描述 */
    message: string,
  ) {
    super(message)
    this.name = 'AppError'
  }
}

// 使用
throw new AppError(404, CODE.AGENT_NOT_FOUND, 'Agent not found')
```

## 服务间错误传播

### server 调用 openclaw 失败时

```ts
/**
 * Core 客户端请求封装
 *
 * 将 openclaw 返回的错误转换为 server 的 AppError，
 * 保留原始错误码和消息。网络错误统一转为 503。
 */
async function coreRequest<T>(path: string, init?: RequestInit): Promise<T> {
  let res: Response
  try {
    res = await fetch(`${OPENCLAW_URL}${path}`, init)
  } catch (err) {
    // 网络错误：openclaw 不可达
    logger.error({ err, path }, 'Core service unreachable')
    throw new AppError(503, CODE.GATEWAY_DISCONNECTED, 'Core service unavailable')
  }

  const body = await res.json()

  if (!body.ok) {
    // 透传 openclaw 的业务错误
    throw new AppError(res.status, body.code, body.message)
  }

  return body.data
}
```

## 前端错误处理

### TanStack Query 全局错误处理

```ts
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      retry: (failureCount, error) => {
        // 4xx 错误不重试
        if (error instanceof ApiError && error.status < 500) return false
        // 最多重试 2 次
        return failureCount < 2
      },
    },
    mutations: {
      // mutation 不自动重试
      retry: false,
    },
  },
})
```

## 进程级错误处理

```ts
/**
 * 捕获未处理的 Promise 拒绝和异常
 *
 * 记录日志后优雅退出，避免进程静默崩溃。
 */
process.on('unhandledRejection', (reason) => {
  logger.fatal({ err: reason }, 'Unhandled rejection')
  process.exit(1)
})

process.on('uncaughtException', (err) => {
  logger.fatal({ err }, 'Uncaught exception')
  process.exit(1)
})
```

## 优雅关闭

```ts
/**
 * 优雅关闭
 *
 * 收到 SIGTERM/SIGINT 信号后：
 * 1. 停止接受新请求
 * 2. 等待进行中的请求完成
 * 3. 关闭数据库连接
 * 4. 断开网关连接
 */
async function gracefulShutdown() {
  logger.info('Shutting down...')

  // 关闭 HTTP 服务
  server.close()

  // 关闭数据库连接池
  await pool.end()

  // 断开网关
  gatewayClient.disconnect()

  logger.info('Shutdown complete')
  process.exit(0)
}

process.on('SIGTERM', gracefulShutdown)
process.on('SIGINT', gracefulShutdown)
```
