# 13 - 测试策略

## 测试框架

| 工具 | 用途 |
|------|------|
| Vitest | 单元测试 + 集成测试（后端 + 前端） |
| Supertest | HTTP 接口测试（模拟请求） |
| Testcontainers | 集成测试中启动真实 PostgreSQL 容器 |
| MSW (Mock Service Worker) | 前端 API mock |

## 测试分层

```
┌─────────────────────────────────┐
│     E2E Tests (后期，可选)        │  ← Playwright，全链路
├─────────────────────────────────┤
│     Integration Tests (重点)     │  ← API 接口 + 数据库 + 服务间调用
├─────────────────────────────────┤
│     Unit Tests (基础)            │  ← 纯逻辑函数、工具函数
└─────────────────────────────────┘
```

**当前阶段重点：集成测试。** 确保 API 接口、数据库操作、服务间调用正确。

## 集成测试环境

### 数据库隔离

使用 Testcontainers 在测试时启动独立的 PostgreSQL 容器，测试结束自动销毁：

```ts
/**
 * 测试环境初始化
 *
 * 启动 PostgreSQL 容器，执行迁移，
 * 每个测试文件使用独立的数据库事务并在结束后回滚。
 */
import { PostgreSqlContainer } from '@testcontainers/postgresql'
import { drizzle } from 'drizzle-orm/node-postgres'
import { Pool } from 'pg'
import { migrate } from 'drizzle-orm/node-postgres/migrator'

let container: StartedPostgreSqlContainer
let pool: Pool

beforeAll(async () => {
  // 启动 PG 容器
  container = await new PostgreSqlContainer('postgres:16-alpine').start()

  pool = new Pool({
    connectionString: container.getConnectionUri(),
  })

  const db = drizzle(pool)

  // 执行迁移
  await migrate(db, { migrationsFolder: './drizzle' })
}, 60_000)

afterAll(async () => {
  await pool.end()
  await container.stop()
})
```

### 每个测试清理数据

```ts
/**
 * 清理所有表数据
 *
 * 在每个测试用例前执行，确保测试之间数据隔离。
 * 使用 TRUNCATE CASCADE 清理所有关联表。
 */
beforeEach(async () => {
  await db.execute(sql`TRUNCATE agents, skills, tasks, audit_logs CASCADE`)
})
```

## 集成测试示例

### Agent API 测试

```ts
import { describe, it, expect, beforeEach } from 'vitest'
import { app } from '../src/app'

describe('Agent API', () => {
  describe('POST /api/agents', () => {
    it('应创建 Agent 并返回 201', async () => {
      const res = await app.request('/api/agents', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${TEST_API_KEY}`,
        },
        body: JSON.stringify({
          name: 'Test Agent',
          emoji: '🤖',
          model: 'gpt-4o',
        }),
      })

      expect(res.status).toBe(201)

      const body = await res.json()
      expect(body.ok).toBe(true)
      expect(body.code).toBe(0)
      expect(body.data.name).toBe('Test Agent')
      expect(body.data.id).toBeDefined()
    })

    it('缺少 name 时应返回 400', async () => {
      const res = await app.request('/api/agents', {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${TEST_API_KEY}`,
        },
        body: JSON.stringify({ emoji: '🤖' }),
      })

      expect(res.status).toBe(400)

      const body = await res.json()
      expect(body.ok).toBe(false)
      expect(body.code).toBe(400)
    })

    it('未认证时应返回 401', async () => {
      const res = await app.request('/api/agents', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ name: 'Test' }),
      })

      expect(res.status).toBe(401)
    })
  })

  describe('GET /api/agents', () => {
    it('应返回 Agent 列表', async () => {
      // 先创建测试数据
      await createTestAgent({ name: 'Agent A' })
      await createTestAgent({ name: 'Agent B' })

      const res = await app.request('/api/agents', {
        headers: { 'Authorization': `Bearer ${TEST_API_KEY}` },
      })

      expect(res.status).toBe(200)

      const body = await res.json()
      expect(body.data.items).toHaveLength(2)
    })
  })
})
```

### 同步引擎测试

```ts
describe('Agent 同步引擎', () => {
  it('应将磁盘发现的 Agent 写入数据库', async () => {
    // 准备：创建临时目录模拟磁盘 Agent
    const tmpDir = await createTempAgentDir({
      name: 'disk-agent',
      identity: '# 🔍 Researcher',
      soul: 'You are a researcher.',
    })

    // 执行同步
    const result = await syncAgents(mockManager, {
      agentsDir: tmpDir,
    })

    // 验证
    expect(result.created).toBe(1)
    const agents = getAllAgents()
    expect(agents).toHaveLength(1)
    expect(agents[0].name).toBe('Researcher')
    expect(agents[0].source).toBe('local')
    expect(agents[0].soul).toBe('You are a researcher.')
  })

  it('dryRun 模式不应写入数据库', async () => {
    const tmpDir = await createTempAgentDir({ name: 'test' })

    const preview = await syncAgents(mockManager, {
      dryRun: true,
      agentsDir: tmpDir,
    })

    expect(preview.added).toHaveLength(1)

    // 数据库应该没有数据
    const agents = getAllAgents()
    expect(agents).toHaveLength(0)
  })
})
```

## 测试覆盖要求

| 模块 | 最低覆盖率 | 说明 |
|------|-----------|------|
| API 路由 | 80% | 每个接口的成功和主要失败场景 |
| 同步引擎 | 90% | 核心逻辑，边界情况多 |
| 事件总线 | 70% | 发布/订阅的基本功能 |
| 工具函数 | 90% | 纯函数，容易测试 |
| 前端组件 | 不强制 | 当前阶段不要求 |
